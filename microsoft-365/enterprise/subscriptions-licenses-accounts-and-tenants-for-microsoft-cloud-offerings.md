# <a name="subscriptions-licenses-accounts-and-tenants-for-microsofts-cloud-offerings"></a>الاشتراكات والتراخيص والحسابات والمستأجرين ل عروض Microsoft السحابية

توفر Microsoft تسلسلا هيكليا من المؤسسات والاشتراكات والتراخيص وحسابات المستخدمين لاستخدام الهويات الفوترة بشكل متناسق عبر عروضها السحابية:
  
- Microsoft 365 Microsoft Office 365
- Microsoft Azure
- Microsoft Dynamics 365

## <a name="elements-of-the-hierarchy"></a>عناصر التسلسل الهيكلي

فيما يلي عناصر التسلسل الهيكلي:
  
### <a name="organization"></a>المؤسسة

تمثل المؤسسة كيانا للأعمال يستخدم عروض Microsoft السحابية، والتي يتم عادة تحديدها بواسطة اسم مجال عام واحد أو أكثر من أسماء مجالات نظام أسماء المجالات (DNS)، مثل contoso.com. المؤسسة هي حاوية للاشتراكات.
  
### <a name="subscriptions"></a>الاشتراكات

الاشتراك هو اتفاقية مع Microsoft لاستخدام نظام أساسي أو خدمات سحابية واحدة من Microsoft أو أكثر، والتي يتم تراكم رسومها استنادا إلى رسوم ترخيص لكل مستخدم أو استهلاك الموارد المستند إلى السحابة. 

- يتم فرض رسوم ترخيص لكل مستخدم على برامج Microsoft كخدمة (SaaS) (Microsoft 365 وديناميكية 365). 
- يتم شحن عروض السحابة (Azure) في النظام الأساسي ل Microsoft كخدمة (PaaS) والبنية الأساسية كخدمة (Azure) استنادا إلى استهلاك الموارد السحابية.
 
يمكنك أيضا استخدام اشتراك تجريبي، ولكن تنتهي صلاحية الاشتراك بعد فترة معينة من الوقت أو رسوم الاستهلاك. يمكنك تحويل اشتراك تجريبي إلى اشتراك مدفوع.
  
يمكن أن يكون لدى المؤسسات اشتراكات متعددة ل عروض Microsoft السحابية. يعرض الشكل 1 مؤسسة واحدة Microsoft 365 اشتراكات متعددة واشتراك Dynamics 365 واشتراكات Azure متعددة.

### <a name="licenses"></a>التراخيص

بالنسبة إلى عروض سحابة SaaS من Microsoft، يسمح الترخيص لحساب مستخدم معين باستخدام خدمات العرض السحابي. يتم فرض رسوم شهرية ثابتة كجزء من اشتراكك. يعين المسؤولون تراخيص إلى حسابات مستخدمين فردية في الاشتراك. على سبيل المثال في الشكل 2، لدى شركة Contoso Corporation اشتراك Microsoft 365 E5 مع 100 ترخيص، مما يسمح بما يصل إلى 100 حساب مستخدم فردي لاستخدام Microsoft 365 E5 والخدمات.
  
**الشكل 2: التراخيص ضمن الاشتراكات المستندة إلى SaaS في مؤسسة**

![مثال على تراخيص متعددة داخل الاشتراكات ل عروض السحابة المستندة إلى SaaS من Microsoft.](../media/Subscriptions/Subscriptions-Fig2.png)

>[!Note]
>من أفضل ممارسات الأمان استخدام حسابات مستخدمين منفصلة تم تعيين أدوار معينة لها للوظائف الإدارية. لا تحتاج حسابات المسؤولين المخصصة هذه إلى تعيين ترخيص للخدمات السحابية التي تديرها. على سبيل المثال، SharePoint مسؤول الحساب أن يتم تعيين ترخيص Microsoft 365 جديد.
>

بالنسبة إلى خدمات السحابة المستندة إلى Azure PaaS، يتم تصميم تراخيص البرامج في أسعار الخدمة.
  
بالنسبة إلى الأجهزة الظاهرية المستندة إلى Azure IaaS، قد تكون هناك حاجة إلى تراخيص إضافية لاستخدام البرنامج أو التطبيق المثبت على صورة جهاز ظاهري. تتضمن بعض صور الأجهزة الظاهرية إصدارات مرخصة من البرامج مثبتة، كما يتم تضمين التكلفة في سعر الدقيقة للخادم. من الأمثلة على ذلك صور الأجهزة الظاهرية SQL Server 2014 SQL Server 2016. 
  
لدى بعض صور الأجهزة الظاهرية إصدارات تجريبية من التطبيقات المثبتة، وهي تحتاج إلى تراخيص تطبيقات برامج إضافية لاستخدامها بعد الفترة التجريبية. على سبيل المثال، تتضمن SharePoint الإصدار التجريبي من الإصدار التجريبي ل Server 2016 إصدارا تجريبيا من SharePoint Server 2016 المثبت مسبقا. لمواصلة استخدام SharePoint Server 2016 بعد تاريخ انتهاء صلاحية الإصدار التجريبي، يجب شراء ترخيص SharePoint Server 2016 وتراخيص العميل من Microsoft. هذه الرسوم منفصلة عن اشتراك Azure ولا يزال معدل الدقيقة لتشغيل الجهاز الظاهري مطبقا.
  
### <a name="user-accounts"></a>حسابات المستخدمين

يتم تخزين حسابات المستخدمين لكل عروض Microsoft السحابية في مستأجر Azure Active Directory (Azure AD)، الذي يحتوي على حسابات المستخدمين ومجموعاتهم. يمكن مزامنة مستأجر Azure AD مع حسابات خدمات مجال Active Directory (AD DS) الموجودة باستخدام Azure AD الاتصال، Windows مستندة إلى الخادم. تعرف هذه باسم مزامنة الدليل.
  
يعرض الشكل 3 مثالا لاشتراكات متعددة في مؤسسة تستخدم مستأجر Azure AD شائعا يحتوي على حسابات المؤسسة.
  
**الشكل 3: اشتراكات متعددة في مؤسسة تستخدم مستأجر Azure AD نفسه**

![مثال مؤسسة ذات اشتراكات متعددة تستخدم مستأجر Azure AD نفسه.](../media/Subscriptions/Subscriptions-Fig3.png)
  
### <a name="tenants"></a>المستأجرون

بالنسبة ل عروض سحابة SaaS، يكون المستأجر هو الموقع الإقليمي الذي يضم الخوادم التي توفر خدمات السحابة. على سبيل المثال، اختارت شركة Contoso Corporation المنطقة الأوروبية لاستضافة اشتراكات Microsoft 365 و EMS و Dynamics 365 الخاصة ب 15000 عامل في المقر الرئيسي في باريس.
  
يمكن أن يكون لخدمات Azure PaaS وأحمال العمل الظاهرية المستندة إلى الأجهزة المستضافة في Azure IaaS إيجار في أي مركز بيانات Azure في جميع أنحاء العالم. يمكنك تحديد مركز بيانات Azure، المعروف بموقعه، عند إنشاء تطبيق Azure PaaS أو خدمة أو عنصر حمل عمل IaaS.
  
إن مستأجر Azure AD هو مثيل محدد من Azure AD يحتوي على حسابات ومجموعات. تتضمن الاشتراكات المدفوعة أو التجريبية Microsoft 365 Dynamics 365 مستأجر Azure AD مجاني. لا يتضمن مستأجر Azure AD هذا خدمات Azure أخرى وهو ليس نفسه الاشتراك التجريبي أو المدفوع من Azure.
  
### <a name="summary-of-the-hierarchy"></a>ملخص التسلسل الهيكلي

فيما يلي مجموعة سريعة:
  
- يمكن أن يكون لدى المؤسسة اشتراكات متعددة
    
  - يمكن أن يكون للاشتراك تراخيص متعددة
    
  - يمكن تعيين التراخيص إلى حسابات المستخدمين الفردية
    
  - يتم تخزين حسابات المستخدمين في مستأجر Azure AD
    
فيما يلي مثال على علاقة المؤسسات والاشتراكات والتراخيص وحسابات المستخدمين:
  
- مؤسسة تم تعريفها حسب اسم مجالها العام.
    
  - اشتراك Microsoft 365 E3 مع تراخيص المستخدمين.
    
    اشتراك Microsoft 365 E5 مع تراخيص المستخدمين.
    
    اشتراك Dynamics 365 مع تراخيص المستخدمين.
    
    اشتراكات Azure متعددة.
    
  - حسابات المستخدمين في المؤسسة في مستأجر Azure AD مشترك.
    
يمكن لعدة اشتراكات تقدمها سحابة Microsoft استخدام مستأجر Azure AD نفسه الذي يعمل كموفر هوية مشترك. يوفر مستأجر Azure AD المركزي الذي يحتوي على الحسابات المتزامنة ل AD DS المحلي هوية مستندة إلى السحابة كخدمة (IDaaS) لمنظمتك. 
  
**الشكل 4: الحسابات المتزامنة والمعرف IDaaS الخاص بالمنظمة**

![الهوية كخدمة (IaaS) IAAS لمنظمتك.](../media/Subscriptions/Subscriptions-Fig4.png)
  
يعرض الشكل 4 كيفية استخدام مستأجر Azure AD شائع من خلال عروض السحابة SaaS وتطبيقات Azure PaaS من Microsoft و الأجهزة الظاهرية في Azure IaaS التي تستخدم Azure AD Domain Services. يتزامن Azure AD الاتصال AD DS المحلية مع مستأجر Azure AD.
  
## <a name="combining-subscriptions-for-multiple-microsoft-cloud-offerings"></a>دمج الاشتراكات ل عروض سحابة Microsoft المتعددة

يصف الجدول التالي كيفية دمج عروض سحابية متعددة من Microsoft استنادا إلى وجود اشتراك لنوع واحد من العروض السحابية (التسميات التي يتم إدخالها أسفل العمود الأول) وإضافة اشتراك لعرض سحابي مختلف (مستعرض عبر الأعمدة).
  
||**Microsoft 365**|**Azure**|**Dynamics 365**|
|:-----|:-----|:-----|:-----|:-----|
|**Microsoft 365** <br/> |NA  <br/> |يمكنك إضافة اشتراك Azure إلى مؤسستك من مدخل Azure.  <br/> |يمكنك إضافة اشتراك Dynamics 365 إلى مؤسستك من مركز مسؤولي Microsoft 365.  <br/> |
|**Azure** <br/> |يمكنك إضافة Microsoft 365 اشتراك إلى مؤسستك.  <br/> |NA  <br/> |يمكنك إضافة اشتراك Dynamics 365 إلى مؤسستك.  <br/> |
|**Dynamics 365** <br/> |يمكنك إضافة Microsoft 365 اشتراك إلى مؤسستك.  <br/> |يمكنك إضافة اشتراك Azure إلى مؤسستك من مدخل Azure.  <br/> |NA  <br/> |
   
هناك طريقة سهلة لإضافة الاشتراكات إلى مؤسستك للخدمات المستندة إلى Microsoft SaaS من خلال مركز الإدارة:
  
1. سجل الدخول إلى مركز مسؤولي Microsoft 365 ([https://admin.microsoft.com](https://admin.microsoft.com)) باستخدام **مسؤول المستخدم** أو **حساب المسؤول** العام.
    
2. من جزء التنقل الأيسر من **الصفحة الرئيسية لمركز** الإدارة، انقر فوق **الفوترة**، ثم **شراء الخدمات**.
    
3. في صفحة **خدمات الشراء** ، اشتر اشتراكاتك الجديدة.
    
يعين مركز الإدارة المؤسسة ومستأجر Azure AD الخاص Microsoft 365 الاشتراك الجديد ل عروض السحابة المستندة إلى SaaS.
  
لإضافة اشتراك Azure مع نفس المؤسسة ومستأجر Azure AD لاشتراكك Microsoft 365:
  
1. سجل الدخول إلى مدخل Azure ([https://portal.azure.com](https://portal.azure.com)) باستخدام Microsoft 365 **Azure AD DC** أو حساب **المسؤول** العام.
    
2. في التنقل الأيسر، انقر فوق **الاشتراكات**، ثم انقر فوق **إضافة**.
    
3. في الصفحة **إضافة اشتراك** ، حدد عرضا وأكمل معلومات الدفع والاتفاقيات.
    
إذا قمت بشراء اشتراكي Azure و Microsoft 365 بشكل منفصل وتريد الوصول إلى مستأجر Azure AD ل Microsoft 365 من اشتراك Azure، فشاهد الإرشادات الموجودة في إضافة اشتراك Azure موجود إلى مستأجر [Azure Active Directory](/azure/active-directory/fundamentals/active-directory-how-subscriptions-associated-directory).
