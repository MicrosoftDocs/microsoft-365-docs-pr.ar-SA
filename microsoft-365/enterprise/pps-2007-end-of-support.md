# <a name="performancepoint-server-2007-end-of-support-roadmap"></a>خارطة طريق انتهاء دعم PerformancePoint Server 2007

*تنطبق هذه المقالة على كل من Microsoft 365 Enterprise Office 365 Enterprise.*

Office 2007 الخوادم والتطبيقات إلى انتهاء الدعم، بما في ذلك الخوادم والتطبيقات التي قد تستخدمها كجزء من حلول المعلومات التجارية (BI). يسرد الجدول التالي تطبيقات المعلومات BI المتأثرة:
  
|**تطبيقات Microsoft BI**|**تاريخ انتهاء الدعم**|
|:-----|:-----|
|ProClarity Analytics Server 6.3 Service Pack 3  <br/> ProClarity Desktop Professional 6.3  <br/> ProClarity SharePoint Viewer 6.3  <br/> |11 يوليو2017  <br/> |
|SharePoint Server 2007 Service Pack 3  <br/> |10 أكتوبر 2017  <br/> |
|PerformancePoint Server 2007 Service Pack 3  <br/> |9 يناير / كانون الثاني 2018  <br/> |
   
لمزيد من المعلومات، راجع الموارد لمساعدتك على الترقية من Office [عملاء وخادمات 2007](upgrade-from-office-2007-servers-and-products.md).
  
## <a name="what-does-end-of-support-mean"></a>ما معنى *انتهاء الدعم* ؟

مثل معظم منتجات Microsoft و PerformancePoint Server 2007 SP3 وبرامج ProClarity و SharePoint Server 2007 SP3، يكون لها دورة حياة الدعم، حيث توفر Microsoft خلالها ميزات جديدة وإصلاحات الأخطاء وتحديثات الأمان. تدوم عادة دورة حياة المنتج لمدة 10 سنوات من الإصدار الأولي للمنتج. تعرف نهاية دورة الحياة هذه ب نهاية دعم المنتج. مع انتهاء دعم ProClarity و PerformancePoint Server و SharePoint Server 2007، لم تعد Microsoft توفر:
  
- الدعم التقني للمشاكل التي قد تحدث.
    
- إصلاحات الأخطاء ل المشاكل التي يتم اكتشافها وقد تؤثر على استقرار الخوادم وإمكانية استخدامها.
    
- إصلاحات الأمان لنقاط الضعف التي يتم اكتشافها وقد تجعل الخوادم أو التطبيقات عرضة للثغرات الأمنية.
    
- تحديثات المنطقة الزمنية.
    
سيستمر تثبيت ProClarity SharePoint Server 2007 SP3 و PerformancePoint Server 2007 SP3 في العمل على الرغم من انتهاء الدعم. ومع ذلك، نوصي بشدة بالترحيل من هذه التطبيقات في أسرع وقت ممكن.

### <a name="help-selecting-a-solution"></a>المساعدة في تحديد حل

مع توفر عدد كبير من خيارات المعلومات المهنية، قد يبدو من الصعب تحديد الخيار الأفضل. يتوفر دليل عبر الإنترنت لمساعدتك. راجع [اختيار أدوات Microsoft Business Intelligence (BI) للتحليل وإعداد التقارير](/sql/reporting-services/choosing-microsoft-business-intelligence-bi-tools-for-analysis-and-reporting).
  
### <a name="what-if-i-dont-upgrade-now"></a>ماذا لو لم أترقية الآن؟

يمكنك اختيار عدم الترقية على الفور. ستستمر الخوادم والتطبيقات الموجودة في التشغيل. ولكنك لن تتلقى أي تحديثات إضافية، بما في ذلك تحديثات الأمان، منذ انتهاء الدعم. وإذا حدث خطأ ما في تطبيقات الخادم، فلن تتمكن من الحصول على المساعدة من الدعم التقني من Microsoft.

### <a name="use-excel-in-the-cloud-or-on-premises"></a>استخدام Excel (في السحابة أو المحلي)

باستخدام Excel، الذي يعرف أيضا باسم *Excel Services* في SharePoint Server، يمكنك عرض المصنفات واستخدامها في نافذة المستعرض، حتى لو لم يكن Excel مثبتا على الكمبيوتر. يمكنك استخدام Excel لإنشاء التقارير، بطاقات الأداء، لوحات المعلومات. بعد ذلك، شارك المصنفات مع الآخرين، الذين يمكنهم استخدام Excel في المستعرض، سواء كانوا يستخدمون SharePoint Online كجزء من Microsoft 365 أو SharePoint Server في الموقع. يمكنك استخدام البيانات المخزنة في الموقع أو في السحابة، مما يمكنك من استخدام مجموعة واسعة من مصادر البيانات.
  
يقارن الجدول التالي المزايا الأساسية لاستخدام Excel مع Microsoft 365 إلى استخدام Excel مع SharePoint Server. يلي المزيد من المعلومات.
  
|**Excel مع Microsoft 365 (في السحابة)**|**Excel مع SharePoint الخادم (المحلي)**|
|:-----|:-----|
|**يمكنك الحصول على أحدث وأعظم إصدار من Excel**. باستخدام Microsoft 365، ستحصل على أحدث إصدار من Excel، الذي يتضمن أنواع مخططات جديدة فعالة، والقدرة على إنشاء المخططات والجداول بسرعة وسهولة، ودعم المزيد من مصادر البيانات. <br/> <br/> **الإعداد أبسط بكثير**. Excel مع Microsoft 365 للأعمال، لذا لا توجد أي أعمال مهمه من جانبك. قم التسجيل ثم تسجيل الدخول، وسوف تعمل بشكل أسرع وأكثر فعالية مما إذا قمت بترقية الخوادم الخاصة بك في الموقع. <br/> <br/> **يمكن للأشخاص الوصول في كل مكان إلى مصنفاتهم**. يمكن للأشخاص عرض المصنفات بأمان من أي مكان، باستخدام الكمبيوتر والهاتف الذكي والكمبيوتر اللوحي. <br/> <br/> **هناك المزيد!** راجع [قدرات المعلومات EXCEL Office 365](https://support.office.com/article/26c0548e-124c-4fd3-aab3-5f64568cb743.aspx). <br/> |**يمكنك إدارة الإعدادات العالمية**. كمسؤول SharePoint، يمكنك تحديد الإعدادات العامة، مثل الأمان وموازنة التحميل وإدارة جلسات العمل وا تخزين المصنف المؤقت واتصالات البيانات الخارجية. <br/> <br/> **يمكنك استخدام Excel Services مع خدمات PerformancePoint**. يمكنك تكوين Excel Services خدمات PerformancePoint كجزء من تثبيت SharePoint Server، Excel Services التقارير في لوحات معلومات PerformancePoint. <br/> <br/> **هناك المزيد!** راجع [المعلومات التجارية في Excel Excel Services (SharePoint Server 2013)](https://support.office.com/article/2740f10c-579d-4b40-a1d9-7beb5d38547c.aspx). <br/> |
   
#### <a name="excel-with-microsoft-365-in-the-cloud"></a>Excel مع Microsoft 365 (في السحابة)

إذا انتقلت إلى Microsoft 365، سيكون لديك أحدث الخدمات والتطبيقات، بما في ذلك Excel 2016. خدمات PerformancePoint غير متوفر في Microsoft 365، لذلك سيتم استبدال محتوى لوحة معلومات PerformancePoint بمحتوى Excel أو تقارير أخرى. الخبر السار هو أن Excel 2016 لديك الكثير من أنواع المخططات الجديدة، ومن الأسهل من أي وقت مضى إنشاء لوحات معلومات مؤثرة في Excel. كما تضاف الميزات الجديدة بشكل منتظم. لمعرفة المزيد، راجع [ما الجديد في Excel 2016 Windows](https://support.office.com/article/5fdb9208-ff33-45b6-9e08-1f5cdb3a6c73.aspx).
  
علاوة على ذلك، إذا قمت بشراء 50 مقعد أو أكثر من Microsoft 365، يمكن لفريق microsoft FastTrack مساعدتك في إعداده. لمعرفة المزيد، تفضل [بزيارة](https://www.microsoft.com/fasttrack/microsoft-365) FastTrack.
  
#### <a name="excel-with-sharepoint-server-on-premises"></a>Excel مع SharePoint الخادم (المحلي)

إذا قمت بالترقية إلى إصدار أحدث من SharePoint، يمكنك استخدام Excel مع Excel Services أو في مستعرض، كما يلي:
  
- Excel Services في SharePoint Server 2010
    
- Excel Services في SharePoint Server 2013
    
- Excel، الذي هو جزء من Office Online Server، مثبتا بشكل منفصل عن SharePoint Server 2016
    
يمكنك تكوين خدمات PerformancePoint الجديد من SharePoint Server أيضا، واستخدامه مع Excel.
  
لمعرفة المزيد حول خيارات SharePoint، راجع SharePoint نهاية دعم [Server 2007](sharepoint-2007-end-of-support.md).
  
لمعرفة المزيد حول Excel Services، راجع Excel Services [العامة (SharePoint Server 2010)](/previous-versions/office/sharepoint-server-2010/ee424405(v=office.14)).
  
### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>استخدام Power BI (في السحابة أو المحلي)

إن Power BI عبارة عن مجموعة من أدوات تحليل الأعمال لتحليل البيانات ومشاركة نتائج التحليلات. باستخدام Power BI، يمكنك استخدام مصادر البيانات في الموقع أو عبر الإنترنت لإنشاء تقارير لوحات معلومات تفاعلية. يمكن للأشخاص عرض التقارير لوحات المعلومات واستخدامها على أجهزة الكمبيوتر أو الأجهزة المحمولة.
  
لا يعد Power BI جزءا من Microsoft 365 أو SharePoint Server. إنه عرض منفصل يتضمن Power BI Desktop وبوابات Power BI وخدمة Power BI. يتكامل Power BI أيضا مع SharePoint Online. يمكنك بدء استخدام Power BI مجانا. استنادا إلى استخدام البيانات واحتياجات الأعمال، يمكنك الترقية لاحقا إلى Power BI Pro باستخدام Microsoft 365 E5. لمعرفة المزيد، راجع [ما هو Power BI؟](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>استخدام Reporting Services (المحلي)

SQL Server Reporting Services حل تقارير قوي. يمكنك تكوين "خدمات إعداد التقارير" في الوضع الأصلي أو في SharePoint المتكامل. يمكنك استخدام العديد من الأدوات المختلفة لكاتب التقارير، بما في ذلك مصمم التقارير مُنشئ التقارير Power View. باستخدام الإصدار الأخير من SQL Server، يمكنك أيضا استخدام SQL Server Mobile Report Publisher لتقديم تقارير ذات مقياس بأي حجم شاشة. يتيح هذا للمستخدمين استخدام التقارير على أجهزتهم المحمولة. لمعرفة المزيد، راجع SQL Server Reporting Services [(SSRS): إنشاء](/sql/reporting-services/create-deploy-and-manage-mobile-and-paginated-reports) تقارير الأجهزة المحمولة والمفبركة بزهاف، ونشرها وإدارتها.
  
### <a name="use-performancepoint-services-on-premises"></a>استخدام خدمات PerformancePoint (المحلي)

تم بيع PerformancePoint Server 2007 بشكل منفصل عن SharePoint Server 2007. بدءا من SharePoint Server 2010، خدمات PerformancePoint هو تطبيق خدمة في SharePoint Server. وبالتالي، لست بحاجة إلى شراء تراخيص خوادم منفصلة أو أجهزة منفصلة لاستخدامها خدمات PerformancePoint.
  
للانتقال من PerformancePoint Server 2007 إلى خدمات PerformancePoint، يمكنك الانتقال إلى إصدار أحدث من SharePoint Server وتكوين خدمات PerformancePoint. يحدد إصدار SharePoint Server الذي تنتقل له ما إذا كان يمكنك استيراد محتوى لوحة المعلومات الموجودة من PerformancePoint Server 2007 إلى خدمات PerformancePoint.
  
- إذا قمت بالترقية إلى SharePoint Server 2010، يمكنك استيراد محتوى لوحة معلومات PerformancePoint من PerformancePoint Server 2007 إلى خدمات PerformancePoint في SharePoint Server 2010. لمعرفة المزيد، راجع [معالج الاستيراد: محتوى PerformancePoint Server 2007 SharePoint Server 2010](/previous-versions/office/sharepoint-server-2010/ee681485(v=office.14)).
    
- إذا انتقلت إلى SharePoint Server 2013 أو SharePoint Server 2016، ستحتاج على المرجح إلى إنشاء محتوى لوحة معلومات جديد (مصادر البيانات والتقارير بطاقات الأداء وصفحات لوحة المعلومات).
    
للبدء في خطة خدمات PerformancePoint الترقية، راجع الموارد التالية:
  
- [SharePoint نهاية دعم خادم 2007](sharepoint-2007-end-of-support.md)
    
- عندما تعرف أي إصدار من SharePoint تنتقل إلى، راجع المقالة المطابقة خدمات PerformancePoint:
    
  - [التخطيط خدمات PerformancePoint (SharePoint Server 2010)](/previous-versions/office/sharepoint-server-2010/ee681486(v=office.14))
    
  - [خدمات PerformancePoint في SharePoint Server 2013](/sharepoint/administration/performancepoint-services-overview)
    
  - [خدمات PerformancePoint في SharePoint Server 2016](/sharepoint/administration/performancepoint-services-overview)
    
عند الترقية إلى خدمات PerformancePoint، ستحصل على العديد من الميزات والتحسينات الجديدة. خدمات PerformancePoint بطاقات أداء محسنة، ومرئيات جديدة، مثل تقرير شجرة التحلل وتفاصيل KPI؛ والمزيد من أنواع المخططات، وإمكانيات تصفية أفضل ل "معلومات الوقت"، وتوافق محسن لإمكانية وصول ذوي الاحتياجات الخاصة. لمعرفة المزيد، راجع [ما الجديد في خدمات PerformancePoint (SharePoint Server 2010)](/previous-versions/office/sharepoint-server-2010/ee661741(v=office.14)).
  
## <a name="where-can-i-get-help-with-my-upgrade"></a>أين يمكنني الحصول على المساعدة في الترقية؟

سواء قمت بالترقية أو الانتقال إلى Microsoft 365، نوصي بالعمل مع شريك Microsoft. يمكن أن يساعدك الشريك المؤهل في تحديد الحل الذي يلبي احتياجات أعمالك ومساعدتك على النشر بشكل أفضل. تفضل بزيارة [مركز شركاء Microsoft](https://go.microsoft.com/fwlink/?linkid=841249)، واستخدام عوامل تصفية البحث للعثور على موفر حلول.
