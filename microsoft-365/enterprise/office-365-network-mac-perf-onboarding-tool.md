# <a name="microsoft-365-network-connectivity-test-tool"></a>Microsoft 365 اختبار اتصال الشبكة

توجد Microsoft 365 اختبار اتصال الشبكة في <https://connectivity.office.com>. إنها أداة مساعد لتقييم الشبكة ورؤى الشبكة المتوفرة في مركز مسؤولي Microsoft 365 ضمن **| قائمة الاتصال**.

> [!IMPORTANT]
> من المهم تسجيل الدخول إلى نطاق المستأجر Microsoft 365 حيث يتم مشاركة كل تقارير الاختبار مع المسؤول وتحميلها إلى المستأجر أثناء تسجيل الدخول.

> [!div class="mx-imgBorder"]
> ![أداة اختبار الاتصال.](../media/m365-mac-perf/m365-mac-perf-test-tool-page.png)

>[!NOTE]
>تدعم أداة اختبار اتصال الشبكة المستأجرين في WW Commercial ولكن سحابة القطاع الحكومي متوسط أو سحابة القطاع الحكومي عالية أو DoD أو الصين.

تستند تحليلات الشبكة في مركز مسؤول Microsoft 365 إلى قياسات منتظمة داخل المنتج لمستأجر Microsoft 365، مجمعة كل يوم. في المقابل، يتم تشغيل تحليلات الشبكة من Microsoft 365 اتصال الشبكة محليا في الأداة.

الاختبار داخل المنتج محدود، ويجمع تشغيل الاختبارات المحلية للمستخدم المزيد من البيانات مما يؤدي إلى الحصول على تحليلات أكثر عمقا. ستظهر تحليلات الشبكة مسؤول Microsoft 365 مركز المعلومات وجود مشكلة في الشبكة في موقع مكتب معين. يمكن Microsoft 365 اختبار الاتصال في تحديد السبب الجذري لهذه المشكلة وتوفير إجراء لتحسين الأداء المستهدف.

نوصي باستخدام هذه الرؤى معا حيث يمكن تقييم حالة جودة الشبكة لكل موقع مكتب في مركز مسؤول Microsoft 365 ويمكن العثور على مزيد من المعلومات المحددة بعد نشر الاختبار استنادا إلى اختبار Microsoft 365 الاتصال.

## <a name="what-happens-at-each-test-step"></a>ما الذي يحدث في كل خطوة اختبار

### <a name="office-location-identification"></a>Office تحديد الموقع

عندما تنقر فوق *الزر تشغيل الاختبار* ، سنعرض صفحة اختبار التشغيل ونحدد موقع المكتب. يمكنك الكتابة في موقعك حسب المدينة والو الولاية والبلد أو اختيار الكشف عنه لك. إذا اكتشفت موقع المكتب، تطلب الأداة خط العرض وطول الخط من مستعرض الويب وتقصر دقته على 300 متر و300 متر قبل الاستخدام. ليس من الضروري تحديد الموقع بدقة أكبر من المبنى لقياس أداء الشبكة.

### <a name="javascript-tests"></a>اختبارات JavaScript

بعد تحديد موقع المكتب، نقوم بتشغيل اختبار زمن زمن نقل TCP في JavaScript ونطلب بيانات من الخدمة حول المستخدمين ونوصي Microsoft 365 خوادم أمامية للخدمة. عند اكتمال هذه الاختبارات، سنعرضها على الخريطة وفي علامة تبويب التفاصيل حيث يمكن عرضها قبل الخطوة التالية.

### <a name="download-the-advanced-tests-client-application"></a>تنزيل تطبيق عميل الاختبارات المتقدمة

بعد ذلك، سنبدأ تنزيل تطبيق عميل الاختبارات المتقدمة. نعتمد على المستخدم لتشغيل تطبيق العميل ويجب أن يكون وقت التشغيل .NET 6.0 مثبتا عليه أيضا.

يوجد جزءان في اختبار اتصال Microsoft 365 الشبكة: <https://connectivity.office.com> موقع الويب وتطبيق عميل Windows قابل للتنزيل يقوم بتشغيل اختبارات متقدمة لاتصال الشبكة. تتطلب معظم الاختبارات تشغيل التطبيق. سيتم ملء النتائج مرة أخرى في صفحة الويب عندما يتم تشغيلها.

وستطلب منك تنزيل تطبيق اختبار العميل المتقدم من موقع الويب بعد اكتمال اختبارات مستعرض الويب. افتح الملف وتشغيله عند مطالبتك بذلك.

> [!div class="mx-imgBorder"]
> ![تطبيق العميل لاختبارات متقدمة.](../media/m365-mac-perf/m365-mac-perf-open-run-file.png)

### <a name="start-the-advanced-tests-client-application"></a>بدء تشغيل تطبيق عميل الاختبارات المتقدمة

بمجرد بدء تشغيل تطبيق العميل، سيتم تحديث صفحة الويب لإظهار هذه النتيجة. سيتم بدء تلقي بيانات الاختبار إلى صفحة الويب. يتم تحديث الصفحة في كل مرة يتم فيها تلقي بيانات جديدة، كما يمكنك مراجعة البيانات عند وصولها.

### <a name="advanced-tests-completed-and-test-report-upload"></a>اكتمال الاختبارات المتقدمة وتحميل تقرير الاختبار

عند اكتمال الاختبارات، ستعرض كل من صفحة الويب والعميل الاختبارات المتقدمة ذلك. إذا قام المستخدم تسجيل الدخول، سيتم تحميل تقرير الاختبار إلى مستأجر العميل.

## <a name="sharing-your-test-report"></a>مشاركة تقرير الاختبار

يتطلب تقرير الاختبار المصادقة لحسابك Microsoft 365 المستخدم. يقوم المسؤول بتحديد كيفية مشاركة تقرير الاختبار.

### <a name="sharing-your-report-with-your-administrator"></a>مشاركة التقرير مع المسؤول

إذا قمت بالتوقيع عند حدوث تقرير اختباري، فيشارك التقرير مع المسؤول.

### <a name="sharing-with-your-microsoft-account-team-support-or-other-personnel"></a>المشاركة مع فريق حساب Microsoft أو الدعم أو الموظفين الآخرين

يتم مشاركة تقارير الاختبار (باستثناء أي تعريف شخصي) مع موظفي Microsoft. يتم تمكين هذه المشاركة بشكل افتراضي ويمكن تعطيلها من قبل المسؤول في "إدارة **| صفحة اتصال الشبكة** في مسؤول Microsoft 365.

### <a name="sharing-with-other-users-who-sign-in-to-the-same-microsoft-365-tenant"></a>المشاركة مع المستخدمين الآخرين الذين سجلوا الدخول إلى المستأجر Microsoft 365 المستأجر

يمكنك اختيار مستخدمين لمشاركة التقرير معهم. يتم تمكين القدرة على الاختيار بشكل افتراضي، ولكن يمكن للمسؤول تعطيلها.

يمكنك مشاركة تقرير الاختبار مع أي شخص من خلال توفير الوصول إلى ارتباط ReportID. ينشئ هذا الارتباط عنوان URL يمكنك إرساله إلى شخص ما بحيث يمكنه إحضار تقرير الاختبار دون تسجيل الدخول. تكون هذه المشاركة معطلة بشكل افتراضي ويجب تمكينها من قبل المسؤول.

### <a name="your-location-information"></a>معلومات موقعك

يعرض هذا القسم نتائج الاختبار المتعلقة بموقعك.

#### <a name="your-location"></a>موقعك

يتم اكتشاف موقع المستخدم من مستعرض ويب المستخدمين. ويمكن أيضا كتابة هذا الخيار حسب اختيار المستخدم. يتم استخدامه لتحديد مسافات الشبكة إلى أجزاء معينة من محيط شبكة المؤسسة. يتم حفظ المدينة من الكشف عن الموقع والمسافة إلى نقاط الشبكة الأخرى فقط في التقرير.

يظهر موقع مكتب المستخدم في طريقة عرض الخريطة.

#### <a name="network-egress-location-the-location-where-your-network-connects-to-your-isp"></a>موقع الخروج من الشبكة (الموقع الذي تتصل فيه شبكتك ب ISP)

نحن نحدد عنوان IP الخاص بتجاوز الشبكة على جانب الخادم. يتم استخدام قواعد بيانات الموقع للبحث عن الموقع التقريبي لمخالفة الشبكة. عادة ما تكون دقة قواعد البيانات هذه حوالي 90٪ من عناوين IP. إذا لم يكن الموقع الذي تم البحث عنه من عنوان IP الخاص بتجاوز الشبكة دقيقا، فقد يؤدي ذلك إلى نتيجة خاطئة. للتحقق من صحة ما إذا كان هذا الخطأ يحدث  عنوان IP معين، يمكنك استخدام مواقع ويب موقع عنوان IP للشبكة يمكن الوصول إليها بشكل عام للمقارنة مع موقعك الفعلي.

#### <a name="your-distance-from-the-network-egress-location"></a>المسافة من موقع الخروج للشبكة

نحن نحدد المسافة من ذلك الموقع إلى موقع المكتب. يتم عرض ذلك كخبرة ثاقبة للشبكة إذا كانت المسافة أكبر من **500** ميل (800 كيلو متر) حيث من المرجح أن يؤدي ذلك إلى زيادة زمن تقدم TCP بأكثر من 25 ثانية وقد يؤثر على تجربة المستخدم.

تعرض الخريطة موقع الخروج للشبكة بالنسبة إلى موقع مكتب المستخدم الذي يشير إلى الشبكة المعاد إدخالها داخل شبكة WAN الخاصة بالمؤسسة.

تنفيذ الخروج من الشبكة المحلية والمباشرة من مواقع مكتب المستخدمين إلى الإنترنت للحصول على Microsoft 365 اتصال الشبكة. تعد التحسينات التي تم إدخالها على الخروج المحلي والمباشرة أفضل طريقة لمعالجة تحليلات الشبكة هذه.

#### <a name="proxy-server-information"></a>معلومات الخادم الوكيل

نحن نحدد ما إذا تم تكوين الخادم (الخوادم) الوكيلة على الجهاز المحلي Microsoft 365 مرور الشبكة في **الفئة تحسين**. نحن نحدد المسافة من موقع مكتب المستخدم إلى الخوادم الوكيلة.

يتم اختبار المسافة أولا بواسطة Ping ICMP. إذا فشل ذلك، فنحن نختبر باستخدام Ping TCP ونبحث أخيرا عن عنوان IP للخادم الوكيل في قاعدة بيانات موقع عنوان IP. نعرض معلومات الشبكة الدقيقة إذا كان الخادم الوكيل أبعد من **500** ميل (800 كيلو متر) بعيدا عن موقع مكتب المستخدم.

#### <a name="virtual-private-network-vpn-you-use-to-connect-to-your-organization"></a>الشبكة الخاصة الظاهرية (VPN) التي تستخدمها للاتصال مؤسستك

يكشف هذا الاختبار ما إذا كنت تستخدم VPN للاتصال بشبكة Microsoft 365. سوف تظهر نتيجة المرور إذا لم يكن لديك VPN، أو إذا كان لديك VPN مع تكوين تقسيم تقسيم مستحسن للانقسام Microsoft 365.

#### <a name="vpn-split-tunnel"></a>VPN Split Tunnel

يتم **اختبار** كل مسار فئة تحسين Exchange Online SharePoint عبر الإنترنت Microsoft Teams لمعرفة ما إذا كان قد تم فحصه على VPN. يتجنب تقسيم حمل العمل VPN بالكامل. يتم إرسال حمل العمل الذي تم تجاوزه عبر VPN. وينقسم حمل العمل المحدد إلى بعض المسارات المرسلة عبر VPN وبعضها منقسم. سوف تظهر نتيجة تمرير ما إذا كانت كل أحمال العمل مقسمة أو انتقائية.

#### <a name="customers-in-your-metropolitan-area-with-better-performance"></a>العملاء في منطقتك الرئيسية مع أداء أفضل

يتم مقارنة زمن زمن تقدم الشبكة بين موقع مكتب المستخدم Exchange Online الخدمة مع عملاء Microsoft 365 في منطقة metro نفسها. تظهر معرفة الشبكة إذا كان أداء 10٪ أو أكثر من العملاء في منطقة Metro نفسها أفضل. وهذا يعني أن المستخدمين سيكون لديهم أداء أفضل في Microsoft 365 المستخدم.

يتم إنشاء تحليلات الشبكة هذه على أساس أن جميع المستخدمين في مدينة ما لديهم حق الوصول إلى البنية الأساسية نفسها للاتصالات والتقارب نفسه مع دوائر الإنترنت وشبكة Microsoft.

#### <a name="time-to-make-a-dns-request-on-your-network"></a>وقت تقديم طلب DNS على شبكتك

يعرض هذا خادم DNS الذي تم تكوينه على جهاز العميل الذي كان يجري الاختبارات. قد يكون خادم DNS Recursive Resolver ولكن هذا أمر شائع. من المرجح أن يكون خادم إعادة توجيه DNS، الذي ي ذاكرة التخزين المؤقت لنتائج DNS ويهيئ أي طلبات DNS لم يتم تخزينها إلى خادم DNS آخر.

يتم توفير هذا للحصول على المعلومات فقط ولا يساهم في أي معرفة ثاقبة للشبكة.

#### <a name="your-distance-from-andor-time-to-connect-to-a-dns-recursive-resolver"></a>المسافة التي تفصلك عن و/أو وقت الاتصال بحل DNS المتكاتف

يتم تعريف "حل DNS التكاتيري" في الاستخدام من خلال تقديم طلب DNS معين ثم طلب عنوان IP من خادم أسماء DNS الذي استلم منه الطلب نفسه. إن عنوان IP هذا هو "حل DNS المت تكراري" وسيبحث عنه في قواعد بيانات موقع عنوان IP للعثور على الموقع. يتم بعد ذلك حساب المسافة من موقع مكتب المستخدم إلى موقع خادم DNS Recursive Resolver. يتم عرض ذلك كرؤية شبكة إذا كانت المسافة أكبر من **500 ميل** (800 كيلو متر).

قد لا يكون الموقع الذي تم البحث عنه من عنوان IP الخاص بتجاوز الشبكة دقيقا وقد يؤدي ذلك إلى نتيجة خاطئة من هذا الاختبار. للتحقق من صحة ما إذا كان هذا الخطأ يحدث لمواقع عناوين IP معينة، يمكنك استخدام مواقع ويب موقع عنوان IP للشبكة يمكن الوصول إليها بشكل عام.

ست تؤثر تحليلات الشبكة هذه تحديدا على تحديد Exchange Online الأمامية للخدمة. لمعالجة هذه المعرفة المتعمقة، يجب أن يكون الخروج المحلي والمباشرة للشبكة أحد المتطلبات الأساسية، ثم يجب أن يكون DNS Recursive Resolver قريبا من تلك الشبكة.

### <a name="exchange-online"></a>Exchange Online

يعرض هذا القسم نتائج الاختبار المتعلقة Exchange Online.

#### <a name="exchange-service-front-door-location"></a>Exchange موقع باب أمامي للخدمة

يتم تعريف Exchange الأمامية للخدمة في Outlook المستخدم بالطريقة نفسها التي يقوم Outlook بذلك ونقيس زمن زمن تقدم TCP للشبكة من موقع المستخدم إلى ذلك. يظهر زمن زمن تقدم TCP ويقارن Exchange الأمامية للخدمة مع قائمة أفضل الأبواب الأمامية للخدمة للموقع الحالي. يتم عرض ذلك كعينة ثاقبة للشبكة إذا لم تكن إحدى أفضل Exchange الخدمة الأمامية (الباب الأمامي) تستخدم.

قد يعود سبب عدم استخدام أحد أفضل Exchange الخدمة الأمامية (الباب الأمامي) إلى عودة الشبكة قبل الخروج من شبكة الشركة وفي هذه الحالة نوصي بخروجه إلى الشبكة المحلية والمباشرة. وقد يحدث ذلك أيضا بسبب استخدام خادم حل DNS بعيد ومتعد، وفي هذه الحالة نوصي بمحاذاة خادم حل DNS الارتدادي مع الخروج من الشبكة.

نقوم بحساب التحسين المحتمل في زمن زمن تقدم TCP (ms) Exchange أمام الخدمة. يتم ذلك من خلال النظر إلى زمن تقدم شبكة موقع مكتب المستخدم الذي تم اختباره وطرح زمن زمن تقدم الشبكة من الموقع الحالي إلى الخزانة Exchange أمام الخدمة. يمثل الفرق الفرصة المحتملة للتحسين.

هذا يسرد أفضل Exchange خدمة أمامية حسب المدينة لموقعك.

#### <a name="service-front-door-recorded-in-the-client-dns"></a>خدمة الباب الأمامي المسجلة في DNS العميل

يعرض هذا اسم DNS وعنوان IP للخادم Exchange الخادم الأمامي للخدمة الذي تم توجيهك له. يتم توفيره للحصول على المعلومات فقط ولا توجد معرفة بالشبكة مقترنة به.

### <a name="sharepoint-online"></a>SharePoint Online

يعرض هذا القسم نتائج الاختبار المتعلقة SharePoint عبر الإنترنت OneDrive.

#### <a name="the-service-front-door-location"></a>موقع باب الخدمة الأمامي

يتم تعريف SharePoint الأمامية للخدمة داخل الخدمة بالطريقة نفسها التي يقوم بها عميل OneDrive ونقيس زمن زمن تقدم TCP للشبكة من موقع مكتب المستخدم إلى ذلك.

#### <a name="download-speed"></a>سرعة التنزيل

نقوم لقياس سرعة التنزيل لملف 15 Mb من SharePoint الأمامية للخدمة. يتم عرض النتيجة بالميغابايت في الثانية للإشارة إلى حجم الملف بالميغابايت الذي يمكن تنزيله من SharePoint أو OneDrive **ثانية واحدة**. يجب أن يكون الرقم مماثلا لعشر الحد الأدنى من النطاق الترددي للدائرة في الميغابت في الثانية. على سبيل المثال، إذا كان لديك اتصال بالإنترنت بدقة 100 ميغابايت في الثانية، فقد تتوقع 10 ميغابايت في الثانية (10 ميغابايت في الثانية).

#### <a name="buffer-bloat"></a>تهتز المخزن المؤقت

أثناء تنزيل 15Mb، نقوم لقياس زمن تقدم TCP إلى SharePoint أمام الخدمة. هذا هو زمن زمن التأخر تحت التحميل ويقارن ب زمن زمن التأخر عندما لا يكون تحت التحميل. غالبا ما تنسب الزيادة في زمن زمن النقل أثناء التحميل إلى المخازن المؤقتة لجهاز شبكة المستهلكين التي يتم تحميلها (أو ازدحمها). يتم عرض معلومات الشبكة لأي زيادة في عددها 1000 أو أكثر.

#### <a name="service-front-door-recorded-in-the-client-dns"></a>خدمة الباب الأمامي المسجلة في DNS العميل

يعرض هذا اسم DNS وعنوان IP للخادم SharePoint الخادم الأمامي للخدمة الذي تم توجيهك له. يتم توفيره للحصول على المعلومات فقط ولا توجد معرفة بالشبكة مقترنة به.

### <a name="microsoft-teams"></a>Microsoft Teams

يعرض هذا القسم نتائج الاختبار المتعلقة Microsoft Teams.

#### <a name="media-connectivity-audio-video-and-application-sharing"></a>اتصال الوسائط (الصوت والفيديو ومشاركة التطبيقات)

يختبر هذا الأمر اتصال UDP بباب Microsoft Teams الأمامية للخدمة. إذا تم حظر هذا، فقد Microsoft Teams استخدام TCP، ولكن سيضعف الصوت والفيديو. اقرأ المزيد حول قياسات شبكة UDP هذه، والتي تنطبق أيضا Microsoft Teams في جودة [الوسائط وأداء اتصال الشبكة في Skype for Business Online](/skypeforbusiness/optimizing-your-network/media-quality-and-network-connectivity-performance).

#### <a name="packet-loss"></a>فقدان الحزم

إظهار فقدان حزمة UDP الذي يتم قياسه في مكالمة صوتية اختبارية 10 ثانية من العميل إلى Microsoft Teams الأمامية للخدمة. يجب أن يكون هذا أقل من **1.00٪** للمرور.

#### <a name="latency"></a>زمن زمن التأخر

إظهار زمن زمن UDP المقيس، الذي يجب أن يكون أقل من **100 ثانية**.

#### <a name="jitter"></a>التهجي

إظهار الفوضى UDP المقاسة، والتي يجب أن تكون أقل من **30 ثانية**.

#### <a name="connectivity"></a>الاتصال

نحن نختبر اتصال HTTP من موقع مكتب المستخدم بجميع نقاط Microsoft 365 الشبكة المطلوبة. يتم نشر هذه في [https://aka.ms/o365ip](./urls-and-ip-address-ranges.md). يتم عرض معرفة الشبكة لأي نقاط نهاية الشبكة المطلوبة، والتي لا يمكن الاتصال بها.

قد يتم حظر الاتصال بواسطة خادم وكيل أو جدار حماية أو جهاز أمان شبكة آخر على محيط شبكة المؤسسة. يتم اختبار الاتصال بميناء TCP 80 مع طلب HTTP، كما يتم اختبار الاتصال بميناء TCP 443 بطلب HTTPS. في حالة عدم وجود استجابة، يتم وضع علامة على FQDN كفشل. إذا كان هناك رمز استجابة HTTP 407، يتم وضع علامة على FQDN كفشل. إذا كان هناك رمز استجابة HTTP 403، فإننا نتحقق من السمة Server للاستجابة وإذا كانت تبدو خادما وكيلا، فإننا نقوم بعلامة فشل هذا الأمر. يمكنك محاكاة الاختبارات التي ننجزها باستخدام Windows سطر الأوامر curl.exe.

نحن نختبر شهادة SSL في كل نقطة Microsoft 365 شبكة مطلوبة في الفئة "تحسين" أو "السماح" كما هو محدد في [https://aka.ms/o365ip](./urls-and-ip-address-ranges.md). إذا لم تعثر أي اختبارات على شهادة Microsoft SSL، فيجب أن يكون قد تم اعتراض الشبكة المشفرة المتصلة بواسطة جهاز شبكة وسيط. تظهر معرفة الشبكة على أي نقاط نهاية شبكة مشفرة تم اعتراضها.

عندما يتم العثور على شهادة SSL لم يتم توفيرها من قبل Microsoft، سنعرض FQDN للاختبار ومالك شهادة SSL للاستخدام. قد يكون مالك شهادة SSL هذا مورد خادم وكيل، أو قد يكون شهادة موقعة ذاتيا للمؤسسة.

#### <a name="network-path"></a>مسار الشبكة

يعرض هذا القسم نتائج تتبع ICMP إلى Exchange Online الأمامية لخدمة SharePoint عبر الإنترنت والباب الأمامي للخدمة عبر الإنترنت Microsoft Teams الأمامية للخدمة. يتم توفيره للحصول على المعلومات فقط ولا توجد معرفة بالشبكة مقترنة به. هناك ثلاثة تتبعات تم توفيرها. تتبع outlook.office365.com، تتبع للعملاء SharePoint الأمامية أو microsoft.sharepoint.com إذا لم يتم توفير واحد، وتتبع _world.tr.teams.microsoft.com._

## <a name="testing-from-the-command-line"></a>الاختبار من سطر الأوامر

نحن نوفر سطر أوامر قابلا للتنفيذ يمكن استخدامه بواسطة أدوات النشر والتنفيذ عن بعد ونجري الاختبارات نفسها المتوفرة في موقع ويب أداة اختبار اتصال Microsoft 365 الشبكة.

يمكن تنزيل أداة اختبار سطر الأوامر هنا: [أداة سطر الأوامر](https://connectivity.office.com/api/AnonymousConnectivityTest/DownloadStandAloneRichClient)

يمكنك تشغيله بالنقر نقرا مزدوجا فوق الملف القابل للتنفيذ في Windows مستكشف الملفات، أو يمكنك تشغيله من موجه أوامر، أو يمكنك جدولته باستخدام مجدول المهام.

في المرة الأولى التي تقوم فيها ب تشغيل القابلة للتنفيذ، سيطلب منك قبول اتفاقية ترخيص المستخدم النهائي (EULA) قبل إجراء الاختبار. إذا سبق لك قراءة EULA وقبلتها، يمكنك إنشاء ملف فارغ يسمى Microsoft-365-Network-Connectivity-Test-EULA-accepted.txt في دليل العمل الحالي للعملية القابلة للتنفيذ عند بدء تشغيلها. لقبول EULA، يمكنك كتابة 'y' واضغط على Enter في نافذة سطر الأوامر عند مطالبتك بذلك.

يقبل القابلة للتنفيذ معلمة سطر الأوامر من /h لإظهار ارتباط إلى وثائق المساعدة هذه.

### <a name="results"></a>النتائج
يتم كتابة إخراج النتائج إلى ملف JSON في مجلد يسمى TestResults الذي تم إنشاؤه في دليل العمل الحالي للعملية ما لم يكن موجودا بالفعل. تنسيق اسم الملف للمخرج هو connectivity_test_result_YYYY-MM-DD-HH-MM-SS.json. تظهر النتائج في عقد JSON التي تتطابق مع الإخراج الموضح على صفحة الويب لموقع ويب Microsoft 365 ويب لأداة اتصال الشبكة. يتم إنشاء ملف نتائج جديد في كل مرة تقوم بتشغيله ولا يقوم الملف القابل للتنفيذ مستقل بتحميل النتائج إلى مستأجر Microsoft لعرضها في صفحات اتصال شبكة مركز الإدارة.

### <a name="launching-from-windows-file-explorer"></a>بدء العمل من Windows مستكشف الملفات
يمكنك ببساطة النقر نقرا مزدوجا فوق القابلة للتنفيذ لبدء الاختبار وستظهر نافذة موجه الأوامر.

### <a name="launching-from-the-command-prompt"></a>بدء من موجه الأوامر
في نافذة CMD.EXE موجه الأوامر، يمكنك كتابة المسار واسم القابل للتنفيذ لتشغيله. اسم الملف Microsoft.Connectivity.Test.exe

### <a name="launching-from-windows-task-scheduler"></a>بدء العمل Windows جدولة المهام
في Windows جدولة المهام، يمكنك إضافة مهمة ل بدء تشغيل الاختبار القابل للتنفيذ بشكل مستقل. يجب تحديد دليل العمل الحالي للمهمة حيث قمت بإنشاء ملف EULA المقبول حيث سيتم حظر القابلة للتنفيذ حتى يتم قبول EULA. لا يمكنك قبول EULA بشكل تفاعلي إذا بدأت العملية في الخلفية بدون وحدة تحكم.

### <a name="more-details-on-the-standalone-executable"></a>مزيد من التفاصيل حول القابلة للتنفيذ مستقلة
تستخدم أداة سطر الأوامر Windows خدمات الموقع للعثور على معلومات بلد ولاية المدينة للمستخدمين لتحديد بعض المسافات. إذا Windows تكون خدمات الموقع معطلة في لوحة التحكم، ستكون التقييمات المستندة إلى موقع المستخدم فارغة. في Windows الإعدادات يجب تشغيل "خدمات الموقع" ويجب أيضا تشغيل "السماح لتطبيقات سطح المكتب بالوصول إلى موقعك".

ستحاول أداة سطر الأوامر تثبيت .NET Framework إذا لم تكن مثبتة بالفعل. كما سيتم تنزيل الاختبار الرئيسي القابل للتنفيذ من أداة Microsoft 365 اختبار اتصال الشبكة و تشغيل ذلك.

## <a name="faq"></a>الأسئلة المتداولة

فيما يلي أجوبة على بعض الأسئلة التي نطرحها بشكل متكرر.

### <a name="what-is-required-to-run-the-advanced-test-client"></a>ما المطلوب لتشغيل عميل الاختبار المتقدم؟

يتطلب عميل الاختبار المتقدم وقت تشغيل .NET 6.0. إذا قمت بتشغيل عميل الاختبار المتقدم بدون ذلك المثبت، سيتم توجيهك إلى [صفحة مثبت .NET 6.0](https://dotnet.microsoft.com/en-us/download/dotnet/6.0/runtime?utm_source=getdotnetcore). تأكد من التثبيت من العمود تشغيل تطبيقات سطح المكتب Windows. يجب أن تكون أذونات المسؤول على الجهاز مطلوبة لتثبيت وقت التشغيل .NET 6.0.

يستخدم عميل الاختبار المتقدم إشارة للتواصل مع صفحة الويب. لذلك، يجب أن تضمن فتح اتصال منفذ TCP 443 **connectivity.service.signalr.net** . لا يتم نشر عنوان URL هذا في لأن <https://aka.ms/o365ip> هذا الاتصال غير مطلوب لمستخدم Microsoft 365 العميل.

### <a name="what-is-microsoft-365-service-front-door"></a>ما هو Microsoft 365 أمام الخدمة؟

إن Microsoft 365 أمام الخدمة هو نقطة دخول على شبكة Microsoft العامة حيث Office العملاء والخدمات اتصال الشبكة. للحصول على اتصال شبكة Microsoft 365، من المستحسن إنهاء اتصال الشبكة إلى أقرب Microsoft 365 أمامي في مدينتك أو في metro.

> [!NOTE]
> Microsoft 365 أي علاقة مباشرة مع منتج **Azure Front Door Service** المتوفر في سوق Azure.

### <a name="what-is-the-best-microsoft-365-service-front-door"></a>ما هو أفضل Microsoft 365 أمامي للخدمة؟

إن أفضل Microsoft 365 أمامية للخدمة (المعروف سابقا بالباب الأمامي للخدمة المثلى) هو الأقرب إلى الخروج من الشبكة، بشكل عام في مدينتك أو منطقة metro. استخدم أداة Microsoft 365 الشبكة لتحديد موقع الباب Microsoft 365 الأمامية للخدمة وأفضل باب أمامي للخدمة. إذا كانت الأداة تحدد أن الباب الأمامي للاستخدام الخاص بك هو أحد أفضلها، يجب أن تتوقع اتصالا رائعا بشبكة Microsoft العامة.

### <a name="what-is-an-internet-egress-location"></a>ما هو موقع الخروج عبر الإنترنت؟

إن موقع خروج الإنترنت هو الموقع الذي يتم فيه خروج حركة مرور الشبكة من شبكة المؤسسة والاتصال بالإنترنت. يتم تعريف هذا أيضا كمقر حيث لديك جهاز ترجمة عنوان الشبكة (NAT) وعادة ما يكون المكان الذي تتصل فيه بموفر خدمة الإنترنت (ISP). إذا رأيت مسافة طويلة بين موقعك وموقع الخروج عبر الإنترنت، فقد يؤدي ذلك إلى تحديد WAN backhaul هام.
