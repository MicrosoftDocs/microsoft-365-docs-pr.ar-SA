# <a name="sharepoint-2007-migration-options-to-consider"></a>SharePoint ترحيل 2007 التي يجب التفكير فيها

*تنطبق هذه المقالة على كل من Microsoft 365 Enterprise Office 365 Enterprise.*

وصلت SharePoint Microsoft 2007 SharePoint Server 2007 إلى نهاية الدعم. حان وقت الترقية! توفر هذه المقالة معلومات حول خيارات الترحيل.
  
## <a name="common-upgrade-strategies-for-sharepoint"></a>استراتيجيات الترقية الشائعة SharePoint

هناك أساليب متعددة لترقية بيئة SharePoint Server. إذا كان لديك مزرعة Microsoft Office SharePoint Server 2007، فيما يلي بعض الأمثلة على أساليب الترقية:
  
- إرفاق قاعدة بيانات
    
- الترقية جنبا إلى جنب
    
- الترقية في مكانها
    
- الترقية المختلطة (في مكان مع قواعد بيانات منفصلة / إرفاق قاعدة بيانات منفصلة)
    
- SharePoint المختلطة (الاتصال عبر الإنترنت بوصلات SharePoint)
    
- نقل البيانات يدويا بين مجموعات المواقع أو المكتبات
    
- FastTrack المعالج إلى Microsoft 365 ([SharePoint نشر عبر الإنترنت](https://aka.ms/spoguidance))
    
- ترحيل API إلى SharePoint Online (SPO) في Microsoft 365
    
ما الأفضل بالنسبة لك؟
  
إن معرفتك بما تقوم به مزرعتك وتستخدمه هي قوة تكتيكية عندما يتعلق الأمر بالترقية. ستساعدك الطريقة التي يستخدم بها الأشخاص SharePoint المزرعة على الاختيار من بين خياراتك.
  
> [!TIP]
> Microsoft Office SharePoint Server 2007 أيضا ترقية تدريجية غير مشمولة هنا. لرؤية قائمة مقالات الترقية الخاصة بخطوة معينة، راجع SharePoint نهاية دعم [Server 2007](sharepoint-2007-end-of-support.md). 
  
تذكر التحقق من دورة [حياة](https://support.microsoft.com/lifecycle/search) المنتج ومتطلبات النظام لأي إصدار SharePoint تقوم بالترقية له. هذا هو الأمر الذي يجب أن تدركه عندما تكون الترقية التالية ضرورية (على سبيل المثال، إذا توقفت مؤقتا عند منتج القديم مثل SharePoint Server 2010 التخطيط لمزيد من الترقيات، تأكد من أنك تعرف تاريخ انتهاء الدعم)، وتأكد من أن لديك أجهزة تدعم خطتك. 
  
إذا كنت تخطط للانتقال إلى بعض مواقع SharePoint أو كلها إلى Microsoft 365 في السحابة، فهذا وقت الإشارة المرجعية إلى ارتباط إلى Microsoft 365 Office 365 [الخدمة.](/office365/servicedescriptions/office-365-service-descriptions-technet-library) ستحتاج إلى أوصاف الخدمة للتعرف على SharePoint عبر الإنترنت وكيفية اختلافها عن الميزات SharePoint Server. ترقية الوظائف Microsoft Office SharePoint مزارع Server 2007. إذا كان التثبيت الخاص بك به مواقع مقطوعة، فصلحها قبل الترقية.
  
## <a name="a-note-about-managing-risk"></a>ملاحظة حول إدارة المخاطر

أساليب مثل 'جنبا إلى جنب' مهمة في نظام منطق الترقية. عند الترقية جنبا إلى جنب، يمكنك الاحتفاظ بمزرعة Microsoft Office SharePoint Server 2007، ولكن يمكنك إنشاء مزرعة في الإصدار التالي منها (SharePoint Server 2010) على الأجهزة الجديدة. يساعد ذلك بثلاث طرق:
  
1. يمكنك أخذ نسخ احتياطية من قواعد بيانات Microsoft Office SharePoint Server 2007 لترقية قواعد البيانات بشكل منفصل، باستخدام إرفاق قاعدة البيانات.
    
2. إذا اكتشفت أن بضع مكتبات مستندات هامة ومعلومات أخرى فقط تستخدم في مزرعة Microsoft Office SharePoint Server 2007، يمكنك اختيار نقل البيانات يدويا من Microsoft Office SharePoint Server 2007 إلى SharePoint  Server 2010، أو يمكنك الوصول فقط إلى الإصدار التالي من مواقع ومواقع ويب معينة (مما يسهل عملك).
    
3. كلما قل ما تفعله بمزرعة خوادم Microsoft Office SharePoint Server 2007، بشكل مباشر، كانت البيانات التي تحتوي عليها المزرعة أكثر أمانا عند الترقية.
    
تعمل أساليب مثل In-Place الترقية مباشرة في مزرعة Microsoft Office SharePoint Server 2007، مما يتيح لك خيارات أقل سهولة لترك مسار والبدء من جديد ببيئة البدائية. يمكنك، قدر الإمكان، إنشاء بعض إجراءات الأمان (مثل أخذ النسخ الاحتياطية من البيئة الأصلية واختبارها). على سبيل المثال، إذا كانت مزرعة Microsoft Office SharePoint Server 2007 ظاهرية، وكانت مكررة لأغراض النسخ الاحتياطي والاستعادة، فعاد إنشاء نسخة احتياطية لقواعد البيانات الأخيرة واستعادتها قبل نافذة الخدمة للترقية. مع العلم أن لديك خيار استعادة النسخ الاحتياطية لقاعدة البيانات لن تمنحك فشلا فحسب، بل من الممكن أن تمنحك التهاني.
  
> [!TIP]
> توجد مستندات أفضل الممارسات للترقية [ل Microsoft Office SharePoint Server 2007](/previous-versions/office/sharepoint-2007-products-and-technologies/cc261992(v=office.12)) و [SharePoint Server 2010](/previous-versions/office/sharepoint-server-2010/cc261992(v=office.14)) و [SharePoint Server 2013](/SharePoint/upgrade-and-update/best-practices-for-upgrading-from-sharepoint-2010-to-sharepoint-2013) [و SharePoint Server 2016](/SharePoint/upgrade-and-update/best-practices-for-upgrade). يمكنك أيضا البحث عن [شركاء Microsoft](https://partnercenter.microsoft.com/pcv/search) الذين لديهم خبرة في استخدام الترقيات أو Microsoft 365 الترحيل. 
  
## <a name="make-your-plan"></a>وضع خطتك

إذا كنت بحاجة إلى الترقية، تحتاج إلى خطة، وحجم واحد لا يناسب الكل في هذه الحالات. قد تكون خطتك بسيطة مثل "إنشاء Microsoft 365 باستخدام SharePoint Online وتسجيل مجال وإعادة توجيه الأشخاص لحفظ ملفاتهم هناك". وقد لا يكون كذلك. هذا القرار قرارك، وهذا ما تحتاج إليه أنت والمستخدمون بالفعل.
  
> [!NOTE]
> من المخاطر أن يتم تشغيل البرنامج الذي انتهت دورة حياته. لن يتم تصحيح المنتجات غير المدعمة بعد الآن عند العثور على المشاكل. وهذا يعني أيضا أنه في حالة وجود تهديدات أمنية جديدة، لن يكون هناك أي تصحيحات أو إصلاحات أمنية لأن منتجات نهاية دورة الحياة لم تعد مدعومة. الرجاء تجنب هذه الحالة! 
  
### <a name="first-know-your-farm"></a>تعرف أولا على المزرعة

عند الترقية، يجب أن يستند اتخاذ القرار إلى ما تقوم به مزرعتك لمنظمتك. ما هي الحاجة التي تلبيها؟ ما هو دوره؟ قد يكون لكل مزرعة في شركتك دورا مختلفا. قد تكون بعض SharePoint الخاصة بك مهمة، وبعضها قد يكون أرشيف ملفات-- هناك للحفظ الآمن. أو، إذا كانت مزرعتك تملأ العديد من الأدوار في وقت واحد، فقد تحتاج إلى معرفة مجموعات المواقع أو الويب أو حتى مكتبات المستندات التي تقوم بها، وأي تخصيصات، وكيفية أهمية هذه المجموعات. قد يبدو تحليل البيانات على هذا المستوى وكأنه عمل كبير، ولكنه يوفر الوقت والجهد لإتقان مجالك قبل الترقية أو الترحيل. بمجرد معرفة كل الأجزاء المتحركة وأهم أجزاء البت، ستعرف أيضا ما تخطته ويمكن أن تتركه خلفك. لن تفيدك هذه المعرفة إلا في المستقبل. 
  
إذا، ما الذي يقوله المستخدمون هو الأكثر أهمية حول مزرعة SharePoint الخادم؟
  
- ميزات SharePoint المضمنة
    
- مجموعة البيانات الكبيرة (مثل أرشيف الملفات)
    
- التوفر
    
- التطبيقات الهامة أو أجزاء الويب أو المستندات في المزرعة (مزرعة المهام الهامة)
    
- معايير التوافق التي تم الالتزام بها
    
- التخصيصات
    
إذا قمت بتشغيل عنصر أساسي في شركتك من مزرعة SharePoint، فقول إنه يعمل ككتالوج كبير من البيانات الهامة حول متطلبات خدمة العميل، يمكنك وضع علامة بجانب "التطبيقات الهامة"، ولكن أيضا "التوفر"-- أي أنه قد يؤثر على عملك إذا لم تتمكن من استخدام SharePoint لبعض الوقت. وبالمثل، يمكنك التحقق من "التخصيصات" لأن الخدمات الهامة التي تقدمها المزرعة تستند إلى التعليمات البرمجية المخصصة أو تعريفات الموقع أو العديد من التخصيصات التي تعمل معا.
  
إذا SharePoint هذه الاحتياجات دون مشاركتك خارج نطاق استخدام ما هو مضمن في البرنامج، وحدثته بشكل عام وانفذت الصيانة والإدارة العادية، فمن الممكن أنك اخترت "مضمن SharePoint"--قد يكون هذا أيضا سبب وجودك في إصدار قديم من SharePoint. بعبارة أخرى، يقوم بالفعل بما تحتاج إليه ولم تكن بحاجة إلى الترقية حتى الآن، في Microsoft Office SharePoint انتهاء الدعم في Server 2007.
  
عندما تقوم بإنشاء قائمة نقطية لهذه الأشياء، يمكنك إنشاء معايير للترقية. بعبارة أخرى، يجب أن تفي أي ترقية بهذا الشريط لكي يتم النظر فيها. يوفر لك ذلك طريقة ستبعد الأساليب التي لا تناسب احتياجاتك حاليا.
  
### <a name="a-simple-sample-plan"></a>نموذج بسيط لخطة

قد تحتاج إلى توافق أوسع مع القيادة وغيرها من المسؤولين على المسار الذي ستتخذه SharePoint الترقية. SharePoint غالبا ما يتعاون مسؤولو الخادم مع Microsoft SQL Server، والعمل مع فرق الشبكات والأمان، والمزيد. في الحالات التي يكون فيها عدد كبير من المساهمين، قد تحتاج إلى إنشاء اتفاقية لخطة الترقية وا الترحيل أو ضبطها. على سبيل المثال، إذا قمت بترحيل البيانات بحيث يستخدم جزء من شركتك SharePoint Online في Microsoft 365، من المرجح أن يكون هناك حاجة إلى ضبط الأداء أو اختباره داخل الشبكة. يجب إعلام الفرق المتأثرة قبل الوقت المحدد.
  
في العينة البسيطة، أعرض SharePoint من قبل المسؤول ثم أعرض الخطة التي وافق عليها جميع المساهمين. للوضوح، وثق الاتفاقيات والقرارات.
  
تبدأ الخطة بعد تحليل معمق للمزرعة، وتحاول تحديد دور المزرعة ونقاط الألم والمعلومات الهامة الأخرى التي ستؤدي إلى تضييق بعض خيارات الترقية. بعد ذلك، يتم تقديم اقتراح ترقية من SharePoint، ويوافق الممساهمين على خطة عمل.
  
قائمة النقطية 'الأكثر أهمية':
  
- التوفر والميزات المضمنة SharePoint والتوافق.
    
- توجد معظم البيانات في ثلاث مجموعات مواقع مشتركة، مع مساحة عمل اجتماع واحدة يستخدمها فريق Dev مهمة وتستخدم بشكل كثيف في مناطق زمنية متعددة حول العالم.
    
- هناك 17 موقعا آخر يتم استخدامها على نطاق واسع.
    
- مكتبتان لمستندات (مساحة عمل الاجتماع والمستندات في مجموعة المواقع مشتركة الجذر) أكبر مكتبتين (أكثر من 8000 مستند لكل منهما). لدينا عدد كبير من المستندات المؤرشفة وقائمة بمرفقات جداول البيانات.
    
- توجد 14 قائمة مكتبات تحتوي على بيانات حساسة يجب أن تظل في "التوافق".
    
- يجب أن تكون لدينا القدرة على القيام باكتشاف إلكتروني أينما انتقلنا.
    
- يجب أن تبقى بعض هذه البيانات في الموقع، وذلك بسبب قواعد InfoSec.
    
 **خيارات الترقية وا الترحيل:**
  
| نعم | لا |
|:-----|:-----|
|ترقية قواعد البيانات مع إرفاق قاعدة البيانات  <br/> |الترقية في مكانها  <br/> |
|الترقية باستخدام مزارع جنبا إلى جنب  <br/> |الترقية المختلطة  <br/> |
|ترحيل API إلى SPO في Microsoft 365 (لبيانات الموقع الشخصية)  <br/> |SharePoint المختلط (غير مطلوب حتى الآن)  <br/> |
|بعض عمليات ترحيل البيانات اليدوية SharePoint عبر الإنترنت للبيانات الهامة  <br/> |FastTrack المعالج إلى Microsoft 365  <br/> |
   
 **خطتي المقترحة:**
  
الترقية المحلية، باستخدام إصدارات SharePoint جنبا إلى جنب، بعضها ظاهري، حتى يمكننا ترقية قواعد البيانات أولا. انتقل من SharePoint 2007 إلى SharePoint 2010. يختبر المسؤولون والمختبرون المزرعة الناتجة. يختبر المستخدمون المزرعة الناتجة. إصلاح أي مشاكل تتعلق بتوقف العرض خلال هذه الفترة. مرة أخرى، جنبا إلى جنب، SharePoint قواعد بيانات 2010 إلى SharePoint 2013. اختبار. اختبار/تجربة المستخدم. إصلاح أي مشاكل تتعلق بتوقف العرض خلال هذه الفترة.
  
- فكر في ما إذا كان بحث مختلط مختلط مع SPO يلبي احتياجاتك.
    
- يمكنك [FastTrack المساعدة](https://fasttrack.microsoft.com) إذا كنت تريد الترقية إلى SharePoint عبر الإنترنت من هنا. 
    
- تحديد ما إذا كان يمكن إلغاء تحميل أي مجموعات مواقع Microsoft 365 اشتراك. (Microsoft 365 التوافق العديد من [معايير التوافق](/compliance/regulatory/offering-home). Microsoft 365 [eDiscovery ويمكنه](https://support.office.com/article/edea80d6-20a7-40fb-b8c4-5e8c8395f6da) القيام [بالتحميس](https://support.office.com/article/A18F8975-AA7F-43B4-A7D6-001D14744D8E) من خلال مركز التوافق.) 
    
وإلا، فتابع الترقية جنبا إلى جنب SharePoint Server 2016.
  
> [!NOTE]
> بين التوصيات التي يجريها المسؤولون الذين يخططون للترقية وبين العملية الفعلية، توجد المحادثات التي تحدث مع مساهمين آخرين تعتمد عليها الترقية. على سبيل المثال، في بعض الأحيان، تجبر الاقتصادات المسؤولين على تغيير خططهم. أيا كان القرار النهائي، يجب عليك توثيق الخطة المتفق عليها، في المستقبل. قد يبدو الأمر كما يلي: 
  
 **خطة الإجراء الخاصة بى:**
  
في الموقع، نستخدم بيئة ظاهرية لإنشاء SharePoint Server 2010 و2013. SharePoint سيتم إنشاء Server 2016 على أجهزة جديدة تلبي متطلبات النظام لعام 2016. سنرفق قواعد البيانات لترقية قواعد البيانات من SharePoint 2007 عبر كل الإصدارات بينها وبين SharePoint Server 2016. يتم إعادة إنشاء التخصيصات الأساسية واختبارها في بيئة SharePoint Server 2016 في الوقت الراهن، إذا لم تكن الميزات الأصلية تلبي احتياجاتنا بالفعل. إذا نجحنا، سيكون لدينا مزرعة على أجهزة جديدة مع قواعد بيانات تمت ترقيتها وتخصيصات أقل. سنقوم بإرفاق قواعد بيانات المحتوى التي تمت ترقيتها إلى مجموعات المواقع الجديدة في SharePoint Server 2013، واختبار واختبار/تجربة المستخدم، ثم إجراء عملية قص DNS إلى بيئة SharePoint Server 2016 الجديدة للاستخدام المباشر.
  
- لن تفكر في استخدام "مختلط مختلط" SharePoint Server 2016 SharePoint Online الآن.
    
- يمكن تحويل ما يقدر بنسبة 35٪ من مواقعنا إلى مواقع SPO جديدة ذات مجالات غير مهيأ، أو، في النهاية، تصبح OneDrive for Business التخزين. البحث عن فرص أخرى لتحويل المواقع أو توجيه مواقع جديدة إلى SPO.
    
- سيكون جزء من هذا الجزء من الترحيل يدويا، عن طريق السحب واسقاط OneDrive for Business الشخصية، والبعض عن طريق ترحيل API.
    
يجب أن تتبع الخطة خطوات أكثر تفصيلا أو عدد من الارتباطات إلى توجيهات ترقية معينة. يجب عدم إلغاء تشغيل كمبيوتر MOSS 2007، كما يجب الاحتفاظ بالبيئات الظاهرية من أجل المقارنة؛ ومع ذلك، ستكتمل الترقية عند إعادة توجيه المستخدمين إلى SharePoint Server 2016.
  
غالبا ما تكون العوامل الرئيسية لاختيار طريقة هي التكلفة الإجمالية للترقية والتكلفة في الوقت المناسب (سترى المزيد حول ذلك في مقالة SharePoint الترحيل). ومع ذلك، فإن التخطيط للمستقبل سيفيدك كثيرا في وضع التوقعات والاختيار بحكمة ووضع إطار لما سيبدو عليه النجاح.
