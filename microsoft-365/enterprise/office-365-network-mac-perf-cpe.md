# <a name="microsoft-365-informed-network-routing"></a>Microsoft 365 توجيه الشبكة المستنير

إن توجيه الشبكة المستنير هو ميزة تدمج تطبيقات Microsoft 365 متعددة مع حلول الشبكة المعرفة من قبل جهة خارجية (SD-WAN) لتحسين اتصال الشبكة بنقاط نهاية خدمة Microsoft وتحسينه. قد يؤدي اتصال SD-WAN المحسن إلى تحسين تجارب المستخدمين وأداءهم.

## <a name="overview"></a>نظرة عامة

يوفر توجيه الشبكة المستنير قناة مشاركة بيانات ثنائية الاتجاه بين Microsoft وحلول SD-WAN. لكل موقع مكتب ودائرة إنترنت تقوم بتكوينها، تشارك Microsoft الملاحظات بشكل دوري مع حل SD-WAN على جودة تجارب تطبيقات Microsoft 365 محددة لحركة مرور الشبكة المقترنة بكل دائرة إنترنت معينة. باستخدام هذه الملاحظات، قد يتخذ حل SD-WAN بعد ذلك إجراءات استرداد ذكية من خلال Microsoft 365 حركة مرور التطبيقات من خلال ارتباطات بديلة متوفرة. 

من الصعب الكشف بشكل مستمر عن انخفاض جودة الخدمة في مسار دائرة إنترنت معينة مثل زيادة زمن زمن الوصول أو فقدان الحزم العالية. قد تكون هذه الانحطاطات ضارة بخبرات المستخدمين في تطبيقات مثل Exchange Online و SharePoint OneDrive و Microsoft Teams. تتضمن الأعراض الشائعة بطء البحث Exchange المحتوى أو أوقات التحويل العالية عند التفاعل مع مكتبات مستندات SharePoint أو OneDrive أو جودة اجتماع أو مكالمة رديئة في Microsoft Teams.

تسعى آلية الاسترداد والملاحظات ضمن توجيه الشبكة المستنير إلى الكشف عن مثل هذه المشاكل بشكل ديناميكي في الوقت الحقيقي تقريبا وإعلام حل SD-WAN الذي تم نشره لاتخاذ إجراءات الاسترداد التلقائي.

يتم أيضا استخدام قناة مشاركة البيانات لتلقي بيانات البصريات على مستوى الشبكة بشكل دوري من حل SD-WAN، بما في ذلك معلومات التكوين وإحصاءات الاستخدام المقترنة للجهاز والدوائر المرفقة. لا يتم تجميع أي معلومات شخصية أو تخزينها. يتم تجميع كل المعلومات التي يتم تجميعها في مواقع Office ودوائر الإنترنت المتصلة. يمكن أن تساعد هذه المعلومات Microsoft على حل المشاكل التي تم Microsoft 365 والخدمات والتطبيقات بطريقة أكثر فعالية وفعالية.

>[!NOTE]
>Microsoft 365 توجيه الشبكة المستنير المستأجرين في سحابة WW التجارية ولكن ليس السحابة سحابة القطاع الحكومي المتوسطة أو سحابة القطاع الحكومي العالية أو DoD أو Germany أو الصين.

## <a name="requirements"></a>المتطلبات

### <a name="integrated-sd-wan-solutions"></a>حلول SD-WAN المتكاملة

تعمل Microsoft مع شركاء مختلفين لتمكين التكامل مع Microsoft 365 توجيه الشبكة المستنير. تتضمن الحلول الممكنة حاليا ما يلي:

| صانع الأجهزة | اسم الحل | الحد الأدنى للإصدار |
| --- | --- | --- |
| Cisco | [IOS XE SD-WAN](https://go.microsoft.com/fwlink/?linkid=2151460) | 20.4/17.4 |

### <a name="network-topology"></a>طبولوجيا الشبكة

يحدد توجيه الشبكة المستنير حاليا حركة المرور المقترنة بموقع مكتب معين ودائرة إنترنت استنادا إلى عنوان IP العام المستخدم لإرسال حركة مرور الشبكة إلى Microsoft. 

في حالة عدم وجود دائرة شبكة واحدة على الأقل توفر اتصالا مباشرا بالإنترنت في موقع فرعي، قد لا يوفر توجيه الشبكة المستنير قيمة مهمة.

### <a name="application-usage"></a>استخدام التطبيق

يتم تجميع بيانات تجربة التطبيق (التي تنعكس من خلال مقاييس جودة الشبكة) من خلال استخدام تطبيقات عميل Microsoft معينة. Exchange المقاييس استخدام Outlook العميل بالإضافة إلى بعض Outlook Web App الاستخدام. SharePoint مقاييس OneDrive استخدام نقاط النهاية الخاصة SharePoint المستأجر، بغض النظر عن تطبيق العميل. Teams المقاييس استخدام عميل سطح المكتب Teams سطح المكتب. لا يتم اعتبار حركة مرور التطبيقات الأخرى عند تقييم صحة دائرة الشبكة.

## <a name="enabling-informed-network-routing"></a>تمكين توجيه الشبكة المستنير

يتطلب تمكين توجيه الشبكة المستنير تنفيذ خطوات متعددة، يجب تنفيذ بعضها ضمن واجهة التكوين الخاصة بالحل SD-WAN. استشر مورد حل SD-WAN للحصول على إرشادات حول كيفية بدء عملية تمكين توجيه الشبكة المستنير داخل حل SD-WAN قبل المتابعة في التكوين في مركز مسؤولي Microsoft 365.

عندما تكون جاهزا لتمكين توجيه الشبكة المستنير في مركز مسؤولي Microsoft 365، تأكد من أن لديك أذونات مسؤول المستخدم أو المسؤول العام الضرورية.

>[!IMPORTANT]
>من أجل توفير الأذونات اللازمة للتطبيقات على مستوى المستأجر، يجب عليك تنفيذ الخطوات التالية كمسؤول عام للوصول إلى قناة مشاركة بيانات توجيه الشبكة المستنيرة.


### <a name="step-1-open-sd-wan-solution-configuration-options"></a>الخطوة 1: فتح خيارات تكوين حل SD-WAN

في [مركز مسؤولي Microsoft 365،](https://admin.microsoft.com/) حدد **> اتصال الشبكة** في جزء التنقل الأيسر.

يوفر هذا القسم من مركز الإدارة مقاييس مجمعة لاتصال الشبكة لمنظمتك وإرشادات حول كيفية تحسين اتصالك. راجع [اتصال الشبكة في مركز مسؤول Microsoft 365 للحصول](office-365-network-mac-perf-overview.md) على معلومات إضافية حول هذه الميزات المتوفرة في مركز الإدارة.

حدد **الإعدادات > SD-WAN لفتح** جزء تكوين توجيه الشبكة المستنير. الخيارات الأخرى التي تظهر ضمن الإعدادات  تنطبق على الإرشادات العامة لاتصال الشبكة في مركز الإدارة ولا تكون مطلوبة لتمكين توجيه الشبكة المستنير.

في جزء التكوين، حدد **إضافة حل SD-WAN**.

### <a name="step-2-select-your-sd-wan-solution-and-data-storage-location"></a>الخطوة 2: تحديد حل SD-WAN وموقع تخزين البيانات

في مربعات المنسدل، حدد حل SD-WAN الذي قمت بنشره والموقع الذي تريد تخزين البيانات المقترنة به مع توجيه الشبكة المستنير. راجع قسم [تخزين البيانات](#data-storage) للحصول على معلومات إضافية.

حدد **التالي**.

### <a name="step-3-accept-terms-for-sharing-of-data"></a>الخطوة 3: قبول شروط مشاركة البيانات

اقرأ جيدا وتأكد من الشروط المتوفرة والمقترنة بمشاركة البيانات بين Microsoft وحلول SD-WAN المحددة، ثم حدد خانة الاختيار المشار إليها.

حدد **التالي**.

### <a name="step-4-grant-permissions-to-the-sd-wan-solution"></a>الخطوة 4: منح الأذونات لحل SD-WAN

ستبدأ هذه الخطوة في طلب منح الأذونات باستخدام Azure Active Directory (Azure AD). سيطلب منك منح أذونات على مستوى المستأجر تسمح بالوصول إلى حل SD-WAN المحدد إلى تخزين بيانات توجيه الشبكة المستنير ومعلومات حماية الخدمة المقترنة بالمستأجر. يتطلب هذا الإجراء **مسؤول Azure AD DC** **أو أذونات** دور المسؤول العام.

حدد الارتباط **منح الإذن لهذا التطبيق** واتبع طلبات Azure AD.

بعد الانتهاء من منح الأذونات، حدد **التالي**.

### <a name="step-5-confirm-your-configuration-settings"></a>الخطوة 5: تأكيد إعدادات التكوين

الخطوة الأخيرة في تمكين توجيه الشبكة المستنير للمستأجر هي صفحة تأكيد تعرض الإعدادات التي قدمتها. 

تم الآن تمكين توجيه الشبكة المستنير للمستأجر.

حدد **تم** ، ثم أغلق جزء تكوين حل SD-WAN.

## <a name="configuring-informed-network-routing"></a>تكوين توجيه الشبكة المستنير

سوف تقوم بإجراء جزء كبير من التكوين من أجل توجيه الشبكة المستنير داخل حل SD-WAN، مثل تكوين كيفية توجيه حركة المرور في الظروف العادية والمسارات البديلة التي يجب استخدامها في حالة اكتشاف مشاكل. استشر موفر حل SD-WAN للحصول على تفاصيل حول خطوات التكوين هذه.

يجب تكوين كل موقع مكتب في مركز مسؤولي Microsoft 365 بحيث يمكن أن يحدد توجيه الشبكة المستنير بشكل صحيح حركة المرور المقترنة بدائرات الشبكة التي توفر الاتصال بهذه المواقع.

Office الكشف التلقائي عن مواقع الشبكة كجزء من مجموعة بيانات الشبكة المستمرة من Microsoft. ونتيجة لذلك، قد يتم ملء بعض المواقع مسبقا في مركز الإدارة للمستأجر. 

إذا كانت هذه المواقع دقيقة، ستحتاج ببساطة إلى تمكين ميزة توجيه الشبكة المستنير لكل موقع تريد وتكوين دوائر الإنترنت وعناوين IP العامة الخاصة بها. 

إذا لم تكن المواقع التي تم الكشف عنها تلقائيا دقيقة، أو إذا لم يكن هناك أي مواقع معبأه مسبقا في المستأجر الخاص بك، يجب إضافة مواقع أو تحريرها يدويا لتعكس طبولوجيا دقيقة لمنظمتك.

### <a name="updating-locations"></a>تحديث المواقع

يمكن العثور على مواقع المستأجر ضمن علامة **التبويب المواقع** . قد يتم تحرير المواقع مباشرة في القائمة أو تحديثها باستخدام ملف CSV.

تأكد من وجود كل موقع مكتب تريد تمكين توجيه الشبكة المستنير فيه في هذه القائمة.

>[!NOTE]
>لا ترتبط الأعمدة الموجودة  في قائمة المواقع للعيينات التي يتم تجميعها والمعلومات الأخرى ذات الصلة بالتقييم بميزة توجيه الشبكة المستنير.

### <a name="enabling-a-location-for-informed-network-routing"></a>تمكين موقع توجيه الشبكة المستنير

1. في القائمة **المواقع** ، حدد **تحرير** من قائمة الإجراءات السريعة لفتح جزء تكوين الموقع.

2. حدد **استخدام Microsoft 365 توجيه الشبكة المستنير في هذا الموقع**.

3. أضف كل دوائر الشبكة التي توفر اتصالا بالإنترنت إلى موقع المكتب هذا في Egress **عناوين IP في قسم موقع** المكتب هذا. تأكد من إقترن كل دائرة بالشبكة الفرعية  عنوان IP العام الفريد الذي يمثل حركة مرور الشبكة.

4. حدد **حفظ** لحفظ التغييرات.

## <a name="disabling-informed-network-routing"></a>تعطيل توجيه الشبكة المستنير

قد يتم تعطيل ميزة توجيه الشبكة المستنير للمستأجر بأكمله عن طريق إعادة تعيين إعدادات حل SD-WAN. على الرغم من أن هذا سيؤدي إلى إيقاف كافة معالجة البيانات ضمن Microsoft 365، يجب أيضا تعطيل توجيه الشبكة المستنير داخل مركز الإدارة.

### <a name="step-1-open-sd-wan-solution-configuration-options"></a>الخطوة 1: فتح خيارات تكوين حل SD-WAN

في [مركز مسؤولي Microsoft 365 حدد](https://admin.microsoft.com/) **> اتصال الشبكة** في جزء التنقل الأيسر.

حدد **الإعدادات > SD-WAN لفتح** جزء تكوين توجيه الشبكة المستنير.

يعرض جزء التكوين ملخصا لحل SD-WAN الذي تم تكوينه حاليا.

### <a name="step-2-reset-your-configuration"></a>الخطوة 2: إعادة تعيين التكوين

في جزء التكوين، حدد **إعادة تعيين إعدادات حل SD-WAN**.

تم الآن إعادة تعيين الإعدادات الخاصة بك، كما تم تعطيل توجيه الشبكة المستنير. يمكنك إعادة تمكينه في أي وقت باتباع الخطوات في [تمكين توجيه الشبكة المستنير](#enabling-informed-network-routing).

## <a name="data-storage"></a>تخزين البيانات

يتم تخزين البيانات التي يتم تبادلها بين Microsoft وموفر حل SD-WAN في موقع تخزين البيانات المحدد أثناء التمكين الأولي توجيه الشبكة المستنيرة. تمثل خيارات موقع تخزين البيانات مناطق جغرافية تحتوي على مناطق Microsoft Azure حيث يتم تخزين البيانات.

يتم الاحتفاظ بالبيانات في هذا الموقع لمدة تصل إلى 30 يوما. عند تعطيلها، تتم إزالة كل البيانات المتبقية ضمن نافذة الاستبقاء هذه التي تستغرق 30 يوما.

يتم تبادل البيانات في هذا الموقع مع حل SD-WAN المحدد، وقد لا يكون موقع حل SD-WAN المكون ضمن المنطقة نفسها. يجب على العملاء العمل مع موفر حلول SD-WAN لتقييم أي متطلبات لموقع تخزين البيانات قبل نشر الإنتاج.
