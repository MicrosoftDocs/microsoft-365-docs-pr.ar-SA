# <a name="information-protection-for-the-contoso-corporation"></a>حماية المعلومات ل Contoso Corporation

شركة Contoso خطيرة بشأن أمان المعلومات. من شأن تسريب الملكية الفكرية أو إتلافها الذي يصف تصميمات منتجاتها وتقنيات التصنيع الخاصة بها أن وضعها في وضع تنافسي غير صالح.

قبل نقل أصولهم الرقمية الحساسة إلى السحابة، تأكد Contoso من أن متطلبات تصنيف المعلومات والحماية المحلية الخاصة بهم مدعومة بواسطة الخدمات المستندة إلى السحابة Microsoft 365 للمؤسسات.

## <a name="the-contoso-path-to-information-protection-with-microsoft-365-for-enterprise"></a>مسار Contoso لحماية المعلومات باستخدام Microsoft 365 للمؤسسة

اتبع Contoso هذه الخطوات لإعداد Microsoft 365 للمؤسسات لمتطلبات حماية المعلومات الخاصة بها:

1. تحديد المعلومات التي يجب حمايتها

   لقد قام Contoso بمراجعة واسعة النطاق للأصول الرقمية الموجودة في مواقع SharePoint والملفات وصنف كل أصل.

2. تحديد سياسات الوصول والاستبقاء وحماية المعلومات لمستويات البيانات

   استنادا إلى مستويات البيانات، حدد Contoso متطلبات النهج المفصلة، التي تم استخدامها لحماية الأصول الرقمية الموجودة عند نقلها إلى السحابة.

3. إنشاء تسميات الحساسية وإعداداتها لمستويات المعلومات المختلفة

   أنشأ Contoso تسميات الحساسية لمستويات البيانات الخاصة به، باستخدام التسمية المنظمة بشكل كبير التي تتضمن التشفير والأذونات العلامات المائية.

4. نقل البيانات من مواقع الويب SharePoint والملفات إلى مواقعها SharePoint الجديدة

    ورثت الملفات التي تم ترحيلها إلى SharePoint الجديدة تسميات الاستبقاء الافتراضية المعينة إلى الموقع.

5. تدريب الموظفين على كيفية استخدام تسميات الحساسية للمستندات الجديدة، وكيفية التفاعل مع Contoso IT عند إنشاء مواقع SharePoint جديدة، وتخزين الأصول الرقمية دائما على مواقع SharePoint

    غالبا ما يعتبر تغيير عادات تخزين المعلومات الخاصة بالعامل السيئ الجزء الأصعب في انتقال حماية المعلومات للسحابة. تحتاج شركة Contoso IT والإدارة إلى دفع الموظفين إلى تسمية أصولهم الرقمية وتخزينها دائما في السحابة، وامتنعوا عن استخدام أسهم الملفات المحلية، وعدم استخدام خدمات التخزين السحابي الخارجية أو محركات أقراص USB.
