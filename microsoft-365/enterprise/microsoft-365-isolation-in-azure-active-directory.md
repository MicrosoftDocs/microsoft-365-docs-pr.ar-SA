# <a name="microsoft-365-isolation-and-access-control-in-azure-active-directory"></a>Microsoft 365 التحكم بالعزل والوصول في Azure Active Directory

تم تصميم Azure Active Directory (Azure AD) لاستضافة مستأجرين متعددين بطريقة آمنة للغاية من خلال عزل البيانات المنطقية. يتم بوابات الوصول إلى Azure AD بواسطة طبقة تخويل. يعزل Azure AD العملاء الذين يستخدمون حاويات المستأجرين لحدود الأمان لحماية محتوى العميل بحيث لا يمكن الوصول إلى المحتوى أو اختراقه بواسطة المستأجرين المشتركين. يتم إجراء ثلاثة تدقيقات بواسطة طبقة تخويل Azure AD:

- هل يتم تمكين الأساسي للوصول إلى مستأجر Azure AD؟
- هل يتم تمكين الأساسي للوصول إلى البيانات في هذا المستأجر؟
- هل دور المدير في هذا المستأجر معتمد لنوع الوصول إلى البيانات المطلوبة؟

لا يمكن لتطبيق أو مستخدم أو خادم أو خدمة الوصول إلى Azure AD بدون المصادقة أو الرمز المميز أو الشهادة المناسبة. يتم رفض الطلبات إذا لم تكن مصحوبة باستخدام بيانات الاعتماد الصحيحة.

بشكل فعال، يستضيف Azure AD كل مستأجر في الحاوية المحمية الخاصة به، مع وجود سياسات وأذونات للحاوية التي يملكها المستأجر ويديرها فقط داخلها.
 
![حاوية Azure.](../media/office-365-isolation-azure-container.png)

مفهوم حاويات المستأجر متعمق بعمق في خدمة الدليل على كل الطبقات، من المداخل إلى التخزين الدائم. حتى عندما يتم تخزين بيانات تعريف مستأجر Azure AD متعددة على القرص الفعلي نفسه، لا توجد علاقة بين الحاويات غير تلك المعرفة بواسطة خدمة الدليل، والتي يتم إملاءها بدورها من قبل مسؤول المستأجر. لا يمكن أن تكون هناك اتصالات مباشرة بمساحة تخزين Azure AD من أي تطبيق أو خدمة تطلبها دون المرور أولا عبر طبقة التفويض.

في المثال أدناه، يحتوي كل من Contoso و Fabrikam على حاويات منفصلة ومخصصة، وعلى الرغم من أن تلك الحاويات قد تشارك بعضا من البنية الأساسية الأساسية نفسها، مثل الخوادم والتخزين، إلا أنها تظل منفصلة ومعزولة عن بعضها البعض، وتم بواباتها بواسطة طبقات من التفويض والتحكم بالوصول.
 
![حاويات Azure المخصصة.](../media/office-365-isolation-azure-dedicated-containers.png)

بالإضافة إلى ذلك، لا توجد مكونات تطبيقات يمكن تنفيذها من داخل Azure AD، ولا يمكن لأحد المستأجرين خرق تكامل مستأجر آخر بشكل جبري، أو الوصول إلى مفاتيح التشفير لمستأجر آخر، أو قراءة البيانات الخام من الخادم.

بشكل افتراضي، لا يحق ل Azure AD جميع العمليات الصادرة عن الهويات في المستأجرين الآخرين. يتم عزل كل مستأجر منطقيا داخل Azure AD من خلال عناصر التحكم بالوصول المستندة إلى المطالبات. يتم تحديد نطاق عمليات قراءة بيانات الدليل وكتابتها في حاويات المستأجرين، كما يتم بواباتها بواسطة طبقة تجريد داخلية وطبقة تحكم بالوصول (RBAC) مستندة إلى الدور، والتي تفرض معا على المستأجر حد الأمان. تتم معالجة كل طلب وصول إلى بيانات الدليل بواسطة هذه الطبقات، ويتم Microsoft 365 كل طلب وصول في Microsoft 365 حسب المنطق أعلاه.

يوجد في Azure AD أقسام أمريكا الشمالية، و"الحكومة الأمريكية"، و"الاتحاد الأوروبي"، و"ألمانيا"، و"العالم". يوجد مستأجر في قسم واحد، ويمكن أن تحتوي الأقسام على مستأجرين متعددين. يتم تجريد معلومات القسم بعيدا عن المستخدمين. يتم نسخ قسم معين (بما في ذلك جميع المستأجرين داخله) إلى مراكز بيانات متعددة. يتم اختيار القسم الخاص ب المستأجر استنادا إلى خصائص المستأجر (على سبيل المثال، رمز البلد). يتم تشفير المعلومات السرية وغيرها من المعلومات الحساسة في كل قسم باستخدام مفتاح مخصص. يتم إنشاء المفاتيح تلقائيا عند إنشاء قسم جديد.

وظائف نظام Azure AD هي مثيل فريد لكل جلسة عمل مستخدم. بالإضافة إلى ذلك، يستخدم Azure AD تقنيات التشفير لتوفير عزل موارد النظام المشتركة على مستوى الشبكة لمنع نقل المعلومات غير المصرح به وغير المقصود.
