# <a name="deploy-a-connector-to-archive-twitter-data"></a>نشر موصل لأرشفة بيانات Twitter

تحتوي هذه المقالة على العملية خطوة بخطوة لنشر موصل يستخدم Office 365 استيراد لاستيراد البيانات من حساب Twitter الخاص بالم مؤسستك Microsoft 365. للحصول على نظرة عامة عالية المستوى حول هذه العملية وقائمة المتطلبات الأساسية المطلوبة لنشر موصل Twitter، راجع إعداد موصل [لأرشفة بيانات Twitter ](archive-twitter-data-with-sample-connector.md).

## <a name="step-1-create-an-app-in-azure-active-directory"></a>الخطوة 1: إنشاء تطبيق في Azure Active Directory

1. انتقل إلى <https://portal.azure.com> سجل الدخول باستخدام بيانات اعتماد حساب مسؤول عام.

   ![سجل الدخول إلى Azure.](../media/TCimage01.png)

2. في جزء التنقل الأيسر، انقر فوق **Azure Active Directory**.

   ![انتقل إلى Azure Active Directory.](../media/TCimage02.png)

3. في جزء التنقل الأيسر، انقر فوق **تسجيلات التطبيق (معاينة)** ثم انقر فوق **تسجيل جديد**.

   ![إنشاء تسجيل تطبيق جديد.](../media/TCimage03.png)

4. تسجيل التطبيق. ضمن **إعادة توجيه URI (اختياري)،** حدد **ويب** `https://portal.azure.com` في القائمة المنسدلة لنوع التطبيق، ثم اكتب المربع الخاص ب URI.

   ![اكتب https://portal.azure.com ل URI إعادة التوجيه.](../media/TCimage04.png)

5. انسخ "معر ة التطبيق" **(العميل)** وملف الدليل **(المستأجر)** واحفظهما في ملف نصي أو موقع آمن آخر. يمكنك استخدام هذه الم IDS في الخطوات اللاحقة.

    ![انسخ "معرّف التطبيق" و"دليل" واحفظه.](../media/TCimage05.png)

6. انتقل إلى **الشهادات & أسرار** التطبيق الجديد، و ضمن **أسرار** العميل، انقر فوق **سري عميل جديد**.

   ![إنشاء سر عميل جديد.](../media/TCimage06.png)

7. إنشاء سر جديد. في مربع الوصف، اكتب السرية ثم اختر فترة انتهاء الصلاحية.

   ![اكتب السرية واختر فترة انتهاء الصلاحية.](../media/TCimage08.png)

8. انسخ قيمة السرية واحفظها في ملف نصي أو موقع تخزين آخر. هذا هو AAD (دليل Azure النشط) سرية التطبيق التي تستخدمها في الخطوات اللاحقة.

   ![انسخ السر واحفظه.](../media/TCimage09.png)


## <a name="step-2-deploy-the-connector-web-service-from-github-to-your-azure-account"></a>الخطوة 2: نشر خدمة الويب للموصل من GitHub إلى حساب Azure

1. انتقل إلى [هذا الموقع GitHub وانقر](https://github.com/microsoft/m365-sample-twitter-connector-csharp-aspnet) **فوق نشر إلى Azure**.

    ![انتقل إلى صفحة Azure الرئيسية.](../media/FBCimage11.png)

2. بعد النقر فوق **نشر إلى Azure**، سيعاد توجيهك إلى مدخل Azure بصفحة قالب مخصصة. قم **بتعبئة التفاصيل** الأساسية **الإعدادات** ثم انقر فوق **شراء**.

   ![انقر فوق إنشاء مورد وا اكتب حساب تخزين.](../media/FBCimage12.png)

    - **الاشتراك:** حدد اشتراك Azure الذي تريد نشر خدمة ويب موصل Twitter له.

    - **مجموعة الموارد:** اختر مجموعة موارد جديدة أو أنشئها. إن مجموعة الموارد هي حاوية تحتوي على موارد ذات صلة لحل Azure.

    - **الموقع:** اختر موقع.

    - **اسم تطبيق ويب:** توفير اسم فريد لتطبيق ويب الموصل. يجب أن يكون طول الاسم بين 3 و18 حرفا. يتم استخدام هذا الاسم لإنشاء عنوان URL لخدمة تطبيق Azure؛ على سبيل المثال، إذا قمت بتوفير اسم تطبيق ويب **ل twitterconnector** ، سيتم عرض عنوان URL **لخدمة تطبيق** Azure twitterconnector.azurewebsites.net.

    - **tenantId:** يتم استخدام "Microsoft 365 المستأجر" الذي نسخته بعد إنشاء تطبيق موصل Facebook في Azure Active Directory في الخطوة 1.

   - **APISecretKey:** يمكنك كتابة أي قيمة كسر. يتم استخدام هذا للوصول إلى تطبيق الويب الموصل في الخطوة 5.

3. بعد نجاح عملية النشر، ستبدو الصفحة مماثلة لشاشة الشاشة التالية:

    ![انقر فوق تخزين، ثم فوق حساب التخزين.](../media/FBCimage13.png)

## <a name="step-3-create-the-twitter-app"></a>الخطوة 3: إنشاء تطبيق Twitter

1. انتقل إلى https://developer.twitter.com، وسجل دخولك باستخدام بيانات الاعتماد لحساب المطور لمنظمتك، ثم انقر فوق **تطبيقات**.

   ![انتقل إلى https://developer.twitter.com وسجل الدخول.](../media/TCimage25-5.png)
2. انقر **فوق إنشاء تطبيق**.

   ![انتقل إلى صفحة التطبيقات لإنشاء تطبيق.](../media/TCimage26.png)

3. ضمن **تفاصيل التطبيق،** أضف معلومات حول التطبيق.

   ![أدخل معلومات حول التطبيق.](../media/TCimage27.png)

4. على لوحة معلومات مطور Twitter، حدد التطبيق الذي أنشأته للتو، ثم انقر فوق **تفاصيل**.

   ![انسخ "معر ة التطبيق" واحفظه.](../media/TCimage28.png)

5. على **علامة التبويب المفاتيح** والرمز المميز، ضمن مفاتيح **API** للمستهلكين، انسخ كل من مفتاح API والمفتاح السري API واحفظهما في ملف نصي أو موقع تخزين آخر. ثم انقر **فوق إنشاء** لإنشاء رمز وصول مميز والوصول إلى سرية الرمز المميز ونسخها إلى ملف نصي أو موقع تخزين آخر.

   ![انسخ المفتاح السري ل API واحفظه.](../media/TCimage29.png)

   ثم انقر **فوق** إنشاء لإنشاء رمز وصول مميز وسر رمز وصول مميز، وانسخها إلى ملف نصي أو موقع تخزين آخر.

6. انقر فوق **علامة التبويب أذونات** وتكوين الأذونات كما هو موضح في لقطة الشاشة التالية:

   ![تكوين الأذونات.](../media/TCimage30.png)

7. بعد حفظ إعدادات الأذونات، انقر فوق **علامة التبويب** تفاصيل التطبيق، ثم انقر فوق تحرير > **تحرير التفاصيل**.

   ![تحرير تفاصيل التطبيق.](../media/TCimage31.png)

8. تنفيذ المهام التالية:

   - حدد خانة الاختيار للسماح لتطبيق الموصل تسجيل الدخول إلى Twitter.

   - أضف OAuth redirect Uri باستخدام التنسيق التالي: **\<connectorserviceuri>/Views/TwitterOAuth**، حيث قيمة *connectorserviceuri* هي URL خدمة تطبيق Azure لم المؤسسة؛ على سبيل https://twitterconnector.azurewebsites.net/Views/TwitterOAuthالمثال، .

    ![اسمح لتطبيق الموصل تسجيل الدخول إلى Twitter وإضافة OAuth redirect Uri.](../media/TCimage32.png)

أصبح الآن تطبيق مطور Twitter جاهزا للاستخدام.

## <a name="step-4-configure-the-connector-web-app"></a>الخطوة 4: تكوين تطبيق ويب الموصل

1. انتقل إلى https://\<AzureAppResourceName>.azurewebsites.net (حيث **AzureAppResourceName** هو اسم مورد تطبيق Azure الذي سميته في الخطوة 4). على سبيل المثال، إذا كان الاسم **twitterconnector،** فاذهب إلى https://twitterconnector.azurewebsites.net. تبدو الصفحة الرئيسية للتطبيق على شكل لقطة الشاشة التالية:

   ![انتقل إلى صفحة موارد تطبيق Azure.](../media/FBCimage41.png)

2. انقر **فوق تكوين** لعرض صفحة تسجيل الدخول.

   ![انقر فوق تكوين لعرض صفحة تسجيل الدخول.](../media/FBCimage42.png)

3. في المربع "تحديد المستأجر"، اكتب "هود المستأجر" أو اللصق (الذي حصلت عليه في الخطوة 2). في مربع كلمة المرور، اكتب APISecretKey (الذي حصلت عليه في الخطوة 2) أو ألصقه، ثم انقر فوق تعيين  تكوين الإعدادات لعرض صفحة تفاصيل التكوين.

   ![سجل الدخول باستخدام مفتاح API ومفتاح API لمفتاح ID المستأجر.](../media/TCimage35.png)

4. أدخل إعدادات التكوين التالية

   - **مفتاح برمجة برمجة مستخدم Twitter:** مفتاح API لتطبيق Twitter الذي أنشأته في الخطوة 3.

   - **مفتاح سري ل Twitter Api:** المفتاح السري ل API لتطبيق Twitter الذي أنشأته في الخطوة 3.

   - **رمز Twitter Access المميز:** رمز الوصول المميز الذي أنشأته في الخطوة 3.

   - **سرية رمز Twitter Access المميز:** سر رمز الوصول المميز الذي أنشأته في الخطوة 3.

   - **AAD (دليل Azure النشط) التطبيق:** "معرّف التطبيق" لتطبيق Azure Active Directory الذي أنشأته في الخطوة 1

   - **AAD (دليل Azure النشط) سرية التطبيق:** قيمة سر APISecretKey الذي أنشأته في الخطوة 1.

5. انقر **فوق** حفظ لحفظ إعدادات الموصل.

## <a name="step-5-set-up-a-twitter-connector-in-the-microsoft-365-compliance-center"></a>الخطوة 5: إعداد موصل Twitter في مركز التوافق في Microsoft 365

1. انتقل إلى مركز التوافق في Microsoft 365، وحدد صفحة <a href="https://go.microsoft.com/fwlink/p/?linkid=2173865" target="_blank">**موصلات** البيانات</a.

2. في صفحة **موصلات البيانات** ضمن **Twitter**، انقر فوق **عرض**.

3. على صفحة **Twitter** ، انقر **فوق إضافة موصل**.

4. في صفحة **شروط الخدمة** ، انقر فوق **قبول**.

5. في الصفحة **إضافة بيانات اعتماد لتطبيق الموصل** ، أدخل المعلومات التالية، ثم انقر فوق **التحقق من صحة الاتصال**.

   ![أدخل بيانات اعتماد تطبيق الموصل.](../media/TCimage38.png)

    - في المربع **الاسم** ، اكتب اسما للموصل، مثل مقبض **تعليمات Twitter**.

    - في المربع **URL الموصل** ، اكتب عنوان URL لخدمة تطبيق Azure أو اللصق فيه؛ على سبيل المثال `https://twitterconnector.azurewebsites.net`.

    - في المربع **كلمة** المرور، اكتب قيمة APISecretKey التي أنشأتها في الخطوة 2 أو قم بلصقها.

    - في المربع **Azure App ID** ، اكتب أو اللصق قيمة Azure Application App Id (يسمى أيضا "اسم العميل *") الذي* حصلت عليه في الخطوة 1.

6. بعد التحقق من صحة الاتصال بنجاح، انقر فوق **التالي**.

7. في الصفحة **تخويل Microsoft 365** البيانات، اكتب APISecretKey أو اللصق مرة أخرى، ثم انقر فوق **تسجيل الدخول إلى تطبيق الويب**.

8. انقر **فوق تسجيل الدخول باستخدام Twitter**.

9. على صفحة تسجيل الدخول إلى Twitter، سجل الدخول باستخدام بيانات الاعتماد لحساب Twitter الخاص مؤسستك.

   ![سجل الدخول إلى حساب Twitter.](../media/TCimage42.png)

   بعد تسجيل الدخول، ستعرض صفحة Twitter الرسالة التالية، "تم إعداد مهمة Twitter Connector بنجاح".

10. انقر **فوق** متابعة لإكمال إعداد موصل Twitter.

11. في الصفحة **تعيين عوامل التصفية** ، يمكنك تطبيق عامل تصفية لاستيراد عناصر ذات عمر معين في البداية. حدد عمرا، ثم انقر فوق **التالي**.

12. في الصفحة **اختيار موقع** التخزين، اكتب عنوان البريد الإلكتروني لعلبة Microsoft 365 البريد التي سيتم استيراد عناصر Twitter لها، ثم انقر فوق **التالي**.

13. انقر **فوق** التالي لمراجعة إعدادات الموصل، ثم انقر فوق **إنهاء** لإكمال إعداد الموصل.

14. في مركز التوافق، انتقل إلى صفحة موصلات البيانات، وانقر فوق علامة التبويب **موصلات** للاطلاع على تقدم عملية الاستيراد.
