# <a name="learn-about-microsoft-feedback-for-your-organization"></a>التعرف على ملاحظات Microsoft لمنظمتك

ملاحظات المستخدمين هي إشارة هامة ل Microsoft لفهم تجارب المستخدم مع منتجات Microsoft. تقدر Microsoft آراء المستخدمين. ملاحظات المستخدمين تذهب مباشرة إلى المهندسين لدينا وتساعدنا على تشكيل مستقبل منتجات وخدمات Microsoft لجميع المستخدمين لدينا.  
في هذا الموضوع، ستجد معلومات حول أنواع الملاحظات الموجودة وكيفية تجميعها وما نجمعه وكيفية معالجة البيانات.

ب أنت مسؤول، يمكنك إدارة إشارة الملاحظات لمنظمتك. نحن نقدم مجموعة جديدة من إعدادات النهج لمساعدتك على إدارة مجموعة ملاحظات المستخدمين ضمن Microsoft 365 تطبيقات مؤسستك. ستساعدك إعدادات النهج هذه على استهداف مجموعات Azure Active Directory وتكوين تجربة مجموعة الملاحظات لم المؤسسة. ملاحظات المستخدمين تذهب مباشرة إلى المهندسين لدينا وتساعدنا على تشكيل مستقبل منتجات وخدمات Microsoft لجميع المستخدمين لدينا. يمكنك معرفة المزيد حول إعدادات النهج هذه والتطبيقات التي تنطبق عليها وأفضل الممارسات في [إدارة ملاحظات Microsoft لمنظمتك](../manage/manage-feedback-ms-org.md).

## <a name="feedback-types"></a>أنواع الملاحظات

### <a name="in-product-feedback"></a>ملاحظات داخل المنتج

إذا كان المستخدمون يستخدمون أحد تطبيقات Microsoft ويرغبون في تقديم الملاحظات، فهناك طرق مختلفة للقيام بذلك من داخل التطبيقات التي يستخدمونها. يمكن للمستخدمين استخدام هذه الطرق المختلفة لمشاركة ملاحظات المنتجات والميزات معنا. إحدى الطرق الأكثر شيوعا لمشاركة الملاحظات عبر تطبيقات Microsoft هي ضمن القائمة تعليمات. عند تحديد **HelpFeedback**  >  من معظم تطبيقات Microsoft، يتم تشغيل صفحة ملاحظات، مما يسمح للمستخدمين بإرسال ملاحظات إلى Microsoft.

### <a name="in-product-surveys"></a>استطلاعات داخل المنتج

يمكن للمستخدمين تصنيف تجربتهم وتوفير معلومات إضافية حول تجربتهم من خلال المطالبات التي بدأها النظام. تحدث هذه المطالبات ضمن Microsoft 365 من وقت لآخر. عند مطالبتك بذلك، يمكن للمستخدمين اختيار ما إذا كانوا يرغبون في تقديم الملاحظات. تظهر عادة المطالبات الخاصة بالمسح في الجزء السفلي الأيمن من التطبيق. إذا قرر المستخدم تقديم الملاحظات أو تجاهل المطالبة أو السماح باختفاء المطالبة من تلقاء نفسه، فلن يرى هذا المستخدم الاستطلاع مرة أخرى لبعض الوقت. تستفيد Microsoft أيضا من عملية إدارة للحد من عدد الاستطلاعات التي بدأها النظام.  الهدف من الإدارة هو ضمان عدم غمر المستخدمين بعدد المطالبات بالمسح.

### <a name="community-feedback"></a>ملاحظات المجتمع

يمكن للمستخدمين الاستفادة من مدخل [ملاحظات Microsoft الجديد](https://feedbackportal.microsoft.com/feedback/) للمشاركة في منتديات المجتمع العام Microsoft 365 المنتجات. سيتيح مدخل الملاحظات الجديد للمستخدمين استعراض الملاحظات التي تم تقديمها بشكل عام وتقديمها بالإضافة إلى إرسال ملاحظات مجتمع عام جديدة لأي من المنتجات المدعومة. تظل أهم عناصر الملاحظات المعروفة متوفرة في المدخل الجديد. هذا المدخل هو أيضا لوحة استجابة جديدة للمستخدمين لتعقب استجابات Microsoft الرسمية والمشاركة مع Microsoft على الملاحظات التي أرسلواها.

## <a name="what-kind-of-feedback-is-best"></a>ما هو أفضل نوع من الملاحظات؟

الملاحظات المفصلة القابلة للتنفيذ ضرورية بإجراء التغييرات والتحسينات في منتجات Microsoft. إذا كان المستخدمون لديك لديهم مشاكل أو اقتراحات حول كيفية تحسيننا، فنحن نريد سماعها. فيما يلي بعض التلميحات والأمثلة حول الملاحظات القابلة للتنفيذ المرسلة إلى Microsoft.

- **عنوان مختصر ووصفي**   تساعدنا العناوين الوصفية والعناوين المحددة على فهم المشكلة التي يتم ا التقارير عنها. مثال: Excel **الملفات** الأخيرة في قائمة الملفات الأخيرة لملفات OneDrive الأخيرة.
- **التركيز على مشكلة واحدة في كل مرة**   تقديم ملاحظات حول مشكلة واحدة أو توصية عنصر واحد في كل مرة. يضمن ذلك تلقي السجلات والبيانات الصحيحة مع كل عملية إرسال ويمكن تعيينها للمتابعة. إذا كان لديك أكثر من مشكلة واحدة، أرسل طلب ملاحظات جديدا لكل مشكلة. يساعدنا هذا في تحديد حجم الملاحظات التي نتلقاها حول مشكلة معينة.
- **كتابة التفاصيل في مربع الوصف**   يتم تضمين معلومات حول جهازك ونظام التشغيل والتطبيقات تلقائيا في كل ملاحظات تم الإعلام عنها. أضف أي معلومات إضافية حول مشكلة تعتقد أنها مهمة. على سبيل المثال، قم بتضمين خطوات مفصلة لإعادة إنتاج المشكلة.

## <a name="how-microsoft-uses-feedback"></a>كيفية استخدام Microsoft الملاحظات

تستخدم Microsoft الملاحظات لتحسين منتجات Microsoft. نحصل على ملاحظات المستخدم في شكل أسئلة ومشاكل ومجاملات واقتراحات. نحن نتأكد من أن هذه الملاحظات تجعلها تعود إلى الفرق المناسبة، التي تستخدم الملاحظات لتحديد منتجات Microsoft وتحديد أولوياتها وتحسيناتها. الملاحظات ضرورية لفرق المنتجات لدينا لفهم تجارب المستخدمين، والتأثير مباشرة على أولوية الإصلاحات والتحسينات.

### <a name="what-do-we-collect"></a>ما الذي نجمعه؟

عندما يقوم المستخدم بإرسال الملاحظات، يتم عادة تجميع معلومات التطبيق مع تصنيفات التطبيقات وأوصاف الملاحظات.  إذا قمت بتمكين النهج، فقد نسمح للمستخدمين بإرسال لقطات الشاشة والسجلات لمساعدتنا على تصحيح المشاكل التي قد يواجهها المستخدم وحلها. فيما يلي العناصر الأكثر شيوعا التي يتم تجميعها أو حسابها.

- **التعليقات**   قام المستخدم بتقديم تعليقات باللغة الأصلية.
- **التطبيق**   لقد تلقينا الملاحظات من منتج Microsoft.
- **تاريخ إرسال**   التاريخ والوقت الذي تلقينا فيه الملاحظات.
- **"اسم المستخدم"**   Azure Active directory Id أو عنوان البريد الإلكتروني للمستخدم المصادق عليه الذي يقوم بتقديم الملاحظات. التعليقات المجهولة مسموحة ولكن غير مبينة في طريقة العرض هذه.
- **البريد الإلكتروني للمستخدم**   إذا كان المستخدم موافقا مع توفير عنوان بريده الإلكتروني للمتابعة.
- **اللغة أو لغة التعليق**   اللغة الأصلية التي تم إرسال التعليق بها.
- **نوع الملاحظات**   ملاحظات الاستطلاع أو الملاحظات داخل التطبيق.
- **أسئلة الاستطلاع**   الأسئلة التي طرحناها على المستخدم أثناء الاستطلاع.
- **استجابات الاستطلاع**   ردود المستخدمين على أسئلة الاستطلاع.
- **قناة**   قناة منتج Microsoft ذات الصلة بالملاحظات.
- **إنشاء التطبيق**   رقم إنشاء منتج Microsoft الذي تم التقاطه عند الإرسال.
- **لغة التطبيق**   لغة منتج Microsoft الذي تم التقاطه عند الإرسال.
- **المرفقات**   كانت أي مرفقات (أي لقطات الشاشة والملفات) التي تم تجميعها كجزء من الملاحظات؟ (نعم/لا).
- **TenantId**   إذا تم إرسال الملاحظات من حساب Azure Active Directory، الذي تم ربطه ب TenantId.
- **الوحدة النمطية للتطبيق** معلومات حول الوحدات النمطية للتطبيق التي ربما تسببت في حدوث عطل حديث، حيثما ينطبق ذلك.

## <a name="how-can-i-see-my-users-feedback"></a>كيف يمكنني رؤية ملاحظات المستخدم؟

لتلبية التزامات Microsoft القانونية تجاه العملاء، نحن نعمل على تجربة جديدة في مركز مسؤولي Microsoft 365 تتيح للمسؤولين عرض بيانات الملاحظات وحذفها وتصديرها لمنظماتهم. وكجزء من مسؤولية وحدة التحكم في البيانات، يمتلك العملاء كل بيانات ملاحظات المستخدمين وستساعد هذه الوظيفة المسؤولين على توفير شفافية مباشرة في تجارب المستخدمين مع منتجات Microsoft 365 وتمكين توفير بيانات ملاحظات المستخدم كجزء من أي طلب موضوع بيانات. سيكون لدى المسؤولين العامين والمسؤولين عن بيانات التوافق القدرة على عرض ملاحظات المستخدمين وتصديرها وحذفها. سيكون جميع المسؤولين الآخرين، بالإضافة إلى القراء، قادرين على عرض بيانات الملاحظات وتصديرها ولكن لا يمكنهم تنفيذ المهام ذات الصلة بالتوافق أو الاطلاع على معلومات حول الأشخاص الذين نشروا الملاحظات (مثل اسم المستخدم أو البريد الإلكتروني أو اسم الجهاز). للوصول إلى بيانات ملاحظات مؤسستك، سجل دخولك إلى مركز مسؤولي Microsoft 365 وخصص التنقل لإظهار عقدة الصحة. يمكنك الوصول إلى هذه التجربة عن طريق تحديد **ملاحظات المنتج** ضمن عقدة "الصحة".

## <a name="data-handling-and-privacy"></a>معالجة البيانات والخصوصية

ندرك أنك عندما تستخدم خدمات السحابة الخاصة بنا، ستوكل إلينا أحد الأصول الأكثر قيمة: بياناتك. نحن نتأكد من تخزين الملاحظات التي نتلقاها والتعامل معها ضمن قواعد إدارة Microsoft، ومن أنه يمكن الوصول إليها فقط الاستخدامات المعتمدة. لا نستخدم البريد الإلكتروني أو الدردشة أو الملفات أو أي محتوى شخصي آخر لاستهداف الإعلانات لك. عندما نجمع البيانات، نستخدمها لتحسين تجاربك.

لمعرفة المزيد حول كيفية حماية خصوصية بياناتك وسريتها، وكيفية ضمان استخدامها فقط بطريقة متسقة مع توقعاتك، راجع مبادئ الخصوصية في [مركز Microsoft Trust Center](https://www.microsoft.com/trust-center/privacy).
